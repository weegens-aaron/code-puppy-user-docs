Understanding Agent Skills

What Are Agent Skills?

Agent Skills are reusable, modular capabilities that extend what Code Puppy agents can do. Think of skills as "plugins" or "modules" that teach agents new specialized tasks.

Why Does It Matter?

Skills allow you to:

  • Extend agent capabilities: Add new functionality without modifying core code
  • Share expertise: Create skills once, use them across multiple projects
  • Customize behavior: Tailor agents to your specific workflow
  • Collaborate: Share skills with your team or the community

How It Works

flowchart LR
    A[Skill Defined] --> B[Skill Discovered]
    B --> C[Skill Activated]
    C --> D[Agent Uses Skill]
    D --> E[Task Completed]

Step-by-step:

  1. Skill Definition: A skill is a folder with a SKILL.md file and optional resources
  2. Skill Discovery: Code Puppy scans configured directories for skills
  3. Skill Activation: Agent loads the skill's instructions when needed
  4. Skill Execution: Agent follows the skill's guidance to complete tasks
  5. Skill Completion: Agent uses the skill's knowledge to produce results

Key Terms

Term Meaning
Skill A reusable capability defined by SKILL.md and resources
SKILL.md The main instruction file for a skill
Skill Directory A folder containing a skill definition
Activate Load a skill's instructions into the agent's context
Resource Supporting files (examples, templates, configs)

Skill Structure

A skill is a folder with this structure:

my-skill/
├── SKILL.md          # Required: Main instructions
├── README.md         # Optional: Documentation
├── examples/         # Optional: Example files
│   ├── example1.py
│   └── example2.py
├── templates/        # Optional: Templates
│   └── template.txt
└── config/           # Optional: Configuration
    └── settings.json

SKILL.md Format

The SKILL.md file defines the skill:

# Skill Name

**Description:** What this skill does

**Tags:** tag1, tag2, tag3

**Version:** 1.0.0

**Author:** Your Name

---

## Overview

Detailed description of what this skill does.

## When to Use

Scenarios where this skill is appropriate.

## How to Use

Step-by-step instructions for using this skill.

## Examples

Concrete examples of the skill in action.

## Resources

Links to additional resources.

Creating Custom Skills

Step 1: Create Skill Folder

mkdir -p ~/.code_puppy/skills/my-custom-skill
cd ~/.code_puppy/skills/my-custom-skill

Step 2: Create SKILL.md

# My Custom Skill

**Description:** A skill that does something useful

**Tags:** custom, utility

**Version:** 1.0.0

**Author:** Your Name

---

## Overview

This skill helps you accomplish [task].

## When to Use

Use this skill when you need to [scenario].

## How to Use

1. Activate the skill
2. Provide your input
3. Follow the guidance

## Examples

Example usage here.

Step 3: Add Resources (Optional)

Add any supporting files your skill needs.

Step 4: Test Your Skill

code_puppy
/activate_skill my-custom-skill

See: Guide: How to create custom skills

Skill Directories

Code Puppy searches for skills in these directories:

Directory Path
User skills ~/.code_puppy/skills/
XDG data $XDG_DATA_HOME/code_puppy/skills/
Project skills .code_puppy/skills/ (project-local)

Configuration:

# In puppy.cfg
skill_directories = ~/.code_puppy/skills
skill_directories = /usr/local/share/code_puppy/skills

Activating Skills

From the CLI

# Activate a skill
/activate_skill my-custom-skill

# List available skills
/list_skills

From an Agent

# Agent can activate skills
result = activate_skill("code-review")

Via REST API

curl -X POST http://localhost:8765/api/commands/execute \
  -H "Content-Type: application/json" \
  -d '{"command": "/activate_skill my-custom-skill"}'

Searching for Skills

By Name

/list_skills python

By Tag

/list_skills security

By Description

/list_skills "code review"

Skill Lifecycle

flowchart TD
    A[Skill Created] --> B[Skill Discovered]
    B --> C{Enabled?}
    C -->|Yes| D[Skill Available]
    C -->|No| E[Skill Disabled]
    D --> F[Skill Activated]
    F --> G[Agent Uses Skill]
    G --> H[Task Completed]
    H --> D

States:

  1. Created: Skill folder exists with SKILL.md
  2. Discovered: Code Puppy found the skill
  3. Available: Skill is enabled and ready
  4. Activated: Skill is loaded in agent context
  5. Disabled: Skill is turned off in config

Common Misconceptions

CAUTION

Misconception: Skills are automatically loaded Reality: Skills must be explicitly activated with /activate_skill

CAUTION

Misconception: All skills are always available Reality: Skills can be disabled in configuration

CAUTION

Misconception: Skills modify code directly Reality: Skills provide guidance; agents execute the changes

Tips

TIP

Use descriptive tags: Tags make skills easier to find

TIP

Include examples: Examples help agents understand how to use the skill

TIP

Version your skills: Track changes with version numbers

TIP

Share skills: Put skills in shared directories for team use

Examples

Example 1: Activating a Skill

You: /activate_skill code-review

Agent: Loading code-review skill...
       Skill activated. Ready to review code.

Example 2: Using a Skill

You: /think Review this file for security issues

Agent: [Uses code-review skill guidance]
       Found 3 potential security issues:
       1. SQL injection vulnerability
       2. Hardcoded credentials
       3. Missing input validation

Example 3: Searching for Skills

You: /list_skills security

Agent: Found 3 skills matching "security":
       1. security-audit - Comprehensive security scanning
       2. secret-detection - Find hardcoded secrets
       3. dependency-check - Check for vulnerable dependencies

Skill Configuration

Enable/Disable Skills

# In puppy.cfg
skills_enabled = true
disabled_skills = old-skill,deprecated-skill

Skill Directories

# Add custom skill directories
skill_directories = ~/.code_puppy/skills
skill_directories = ./project-skills
