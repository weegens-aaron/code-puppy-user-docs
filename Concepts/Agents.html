Understanding Agents

What Is an Agent?

An agent in Code Puppy is an autonomous AI assistant that can perform tasks on your behalf. Think of agents as specialized workers, each with their own expertise and capabilities.

Why Does It Matter?

Agents are the core of Code Puppy's functionality. They:

  • Execute tasks: Run commands, edit files, search codebases
  • Make decisions: Choose which tools to use and when
  • Learn from context: Remember previous interactions and adapt
  • Work collaboratively: Can invoke other agents for specialized work

How It Works

flowchart TB
    A[You give a task] --> B[Agent receives prompt]
    B --> C[Agent plans approach]
    C --> D[Agent uses tools]
    D --> E[Agent produces result]
    E --> F[You get the answer]

Step-by-step:

  1. You provide a task: Give the agent a prompt describing what you want
  2. Agent analyzes: The agent reads your prompt and understands the goal
  3. Agent plans: The agent decides which tools to use and in what order
  4. Agent executes: The agent uses tools (read files, run commands, etc.)
  5. Agent reports: The agent provides you with the result

Key Terms

Term Meaning
Agent An AI worker that performs tasks
Prompt The task description you give to an agent
Tool A capability the agent can use (read files, run commands, etc.)
Session A conversation history with an agent
Model The AI model that powers the agent

Available Agents

Code Puppy comes with several built-in agents:

code-puppy (Default)

Description: The main Code Puppy agent for general-purpose coding tasks.

Capabilities:

  • Code review and analysis
  • Refactoring and optimization
  • Debugging and troubleshooting
  • File operations and search

Pack Agents

When enable_pack_agents = true, additional specialized agents are available:

Agent Role Best For
pack-leader Coordinator Orchestrating complex workflows
bloodhound Searcher Finding patterns and information
husky Heavy lifter Resource-intensive tasks
shepherd Workflow manager Multi-step processes
terrier Small tasks Quick, focused operations
watchdog Monitor Continuous monitoring and alerts
retriever Information fetcher Gathering data from multiple sources

Creating Custom Agents

You can create your own agents with custom capabilities:

# In a plugin or custom agent file
from code_puppy.agents import register_agent

def my_custom_agent():
    return {
        "name": "my-specialist",
        "description": "Specialized for database migrations",
        "capabilities": ["sql-analysis", "migration-planning"]
    }

register_agent("my-specialist", my_custom_agent)

See: Guide: How to create custom agents

Agent Sessions

What Is a Session?

A session is a complete conversation with an agent, including:

  • The initial prompt
  • All messages exchanged
  • Tools used and their results
  • The final outcome

Session Management

Save a session:

/save my-session

List saved sessions:

/sessions list

Load a session:

/sessions load my-session

See: Reference: Session Commands

Choosing the Right Agent

For General Tasks

Use the default code-puppy agent. It's versatile and handles most coding tasks well.

For Specialized Work

Consider using pack agents:

Task Type Recommended Agent
Code search bloodhound
Complex refactoring pack-leader + husky
Monitoring watchdog
Quick fixes terrier

Agent Configuration

Agents inherit settings from your configuration:

# In puppy.cfg
model = gpt-5
temperature = 0.7

Per-agent settings:

# Different model for specific agent
model_settings_code_puppy_temperature = 0.7
model_settings_bloodhound_temperature = 0.5

See: Reference: Configuration Options

Agent Invocation

From the CLI

# Start Code Puppy
code_puppy

# The default agent starts automatically

From Another Agent

# Agents can invoke other agents
result = invoke_agent(
    agent_name="bloodhound",
    prompt="Find all TODO comments in src/"
)

Via REST API

curl -X POST http://localhost:8765/api/agents/invoke \
  -H "Content-Type: application/json" \
  -d '{
    "agent": "code-puppy",
    "prompt": "Review this code",
    "model": "gpt-5"
  }'

See: Reference: REST API

Agent Lifecycle

flowchart TD
    A[Agent Created] --> B[Session Started]
    B --> C[Task Assigned]
    C --> D[Tools Used]
    D --> E[Result Produced]
    E --> F[Session Saved]
    F --> G[Agent Idle]
    G --> B

States:

  1. Created: Agent is initialized and ready
  2. Active: Agent is working on a task
  3. Idle: Agent is waiting for a new task
  4. Completed: Agent finished and session is saved

Common Misconceptions

CAUTION

Misconception: Agents are always online Reality: Agents only run when you invoke them or when scheduled tasks are active

CAUTION

Misconception: All agents have the same capabilities Reality: Different agents may have different tool access and specializations

CAUTION

Misconception: Agents remember everything forever Reality: Each session is independent unless you explicitly save and load it

Tips

TIP

Use descriptive prompts: The more specific your task description, the better the agent can help

TIP

Save important sessions: Use /save to preserve useful conversations for later

TIP

Try different agents: If one agent isn't giving good results, try a different specialized agent

TIP

Monitor agent usage: Check the scheduler and session logs to see what agents are doing

Examples

Example 1: Basic Code Review

You: /think Review this file for security issues
Agent: [Reads file, analyzes code, provides findings]

Example 2: Multi-Agent Workflow

You: /think Find all API endpoints and document them

Agent (code-puppy):
  → Invokes bloodhound to find endpoint definitions
  → Receives list of endpoints
  → Creates documentation
  → Returns final document

Example 3: Scheduled Task

# Create a daily task
/scheduler create --name "daily-check" \
  --schedule "0 9 * * *" \
  --agent "watchdog" \
  --prompt "Check for security vulnerabilities"
