Understanding Code Puppy's Privacy Model

What Is the Privacy Model?

Code Puppy's privacy model is a zero-compromise architecture that ensures your code, conversations, and project data never leave your control unless you explicitly choose to send them to an LLM provider.

Unlike commercial AI coding tools, Code Puppy is built from the ground up with privacy as a fundamental requirement—not an optional feature.

Why Does It Matter?

When you're working on proprietary code, sensitive projects, or regulated systems, you need to know:

  • Your code stays yours: No third party can access what you're building
  • No hidden tracking: Your usage patterns aren't being harvested for advertising
  • Full control: You decide what data leaves your machine and where it goes
  • Auditability: Everything is open source—you can verify the claims

For regulated environments: Healthcare (HIPAA), finance (SOC 2), government contracts, or any scenario where data sovereignty matters.

How It Works (Simplified)

flowchart LR
    A[You run Code Puppy] --> B[Local processing]
    B --> C{Choose model}
    C -->|Local model| D[Zero data leaves network]
    C -->|Cloud model| E[Only prompt sent to provider]
    E --> F[Response returned]
    F --> G[Result shown locally]

The data flow:

  1. You type a prompt in the Code Puppy terminal
  2. Code Puppy processes it locally (no external calls yet)
  3. You choose a model (local or cloud provider)
  4. If local model: Everything stays on your machine
  5. If cloud model: Only your prompt goes to the provider
  6. Response returns and is displayed to you
  7. No logs, no storage, no tracking

Term Meaning

Local model An AI model running on your own hardware or network
Cloud provider External API like OpenAI, Anthropic, etc.

What Code Puppy Does NOT Collect

❌ Zero Telemetry

Code Puppy does not send:

  • Usage statistics (how often you use it, what features you use)
  • Crash reports or error logs to external servers
  • Performance metrics or timing data
  • Feature adoption tracking
  • A/B testing or experimentation data

❌ Zero Prompt Logging

Code Puppy does not store:

  • Your code snippets or file contents
  • Your conversation history with agents
  • Your project structure or file names
  • Your prompts or agent responses
  • Any identifying information about your work

❌ Zero Behavioral Profiling

Code Puppy does not track:

  • When you use the tool
  • How long you work on tasks
  • What types of projects you build
  • Your coding patterns or style
  • Your productivity metrics

❌ Zero Third-Party Sharing

Code Puppy does not share:

  • Your data with investors or partners
  • Your information with analytics platforms
  • Your usage data with advertisers
  • Any information with external services (except the LLM provider you choose)

Data Flow by Model Type

Local Models (Maximum Privacy)

When you use a local model (VLLM, Ollama, Llama.cpp, etc.):

flowchart LR
    A[Your Terminal] --> B[Code Puppy]
    B --> C[Local Model Server]
    C --> B
    B --> A
    style A fill:#90EE90
    style B fill:#90EE90
    style C fill:#90EE90

Result: Zero data leaves your network. Everything runs on your hardware.

Setup: Configure in ~/.code_puppy/extra_models.json:

{
  "local-codestral": {
    "type": "openai",
    "name": "codestral-latest",
    "custom_endpoint": {
      "url": "http://localhost:8080/v1",
      "api_key": "local-key"
    }
  }
}

Cloud Models (Provider Privacy)

When you use a cloud provider (OpenAI, Anthropic, etc.):

flowchart LR
    A[Your Terminal] --> B[Code Puppy]
    B --> C[LLM Provider API]
    C --> B
    B --> A
    style A fill:#90EE90
    style B fill:#90EE90
    style C fill:#FFB6C1

Result: Your prompt goes to the provider, response returns. Code Puppy never sees or stores it.

Important: Each provider has their own privacy policy. Code Puppy can't control what they do with your data—only that we don't add to it.

Our Privacy-First Architecture

Structural Enforcement

NOTE

Our privacy commitment is structurally enforced—it's not just a policy, it's built into the code.

How we enforce it:

  1. No external API calls except to your chosen LLM provider
  2. No analytics SDKs or tracking libraries included
  3. No cloud storage for user data or logs
  4. Open source codebase—anyone can audit what we do
  5. MIT license—no hidden terms or conditions

The "No Investor Pressure" Advantage

Unlike venture-backed competitors, Code Puppy has:

  • No investors demanding growth metrics
  • No board pushing for monetization through data
  • No quarterly earnings to report
  • No acquisition that could change privacy policies

This means privacy isn't a feature we might compromise for profit—it's a principle we can maintain indefinitely.

Common Misconceptions

CAUTION

Misconception: "All AI coding tools are the same" Reality: Commercial tools often log prompts, track usage, and may use your data for training. Code Puppy does none of these.

CAUTION

Misconception: "Privacy means less functionality" Reality: Code Puppy has full-featured agents, tools, and capabilities—privacy doesn't require sacrificing features.

CAUTION

Misconception: "I need to trust Code Puppy's privacy claims" Reality: Everything is open source. You can audit the code yourself to verify no data exfiltration.

CAUTION

Misconception: "Local models are too slow" Reality: Modern local models on decent hardware can be very fast, and you control the trade-off between speed and privacy.

Tips

TIP

Use local models for maximum privacy: Set up VLLM, Ollama, or Llama.cpp on your machine for zero data egress.

TIP

Audit the code yourself: Clone the repo and grep for network calls to verify what data leaves your machine.

TIP

Check your firewall: Monitor outbound connections to verify only your chosen LLM provider is contacted.

TIP

Review provider policies: When using cloud models, read the privacy policy of your chosen provider (OpenAI, Anthropic, etc.).

TIP

Use environment variables: Store API keys in environment variables, not in config files, for better security.
