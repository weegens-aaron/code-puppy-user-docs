Frequently Asked Questions

Getting Started

How do I install Code Puppy?

The recommended way is using uv:

uvx code-puppy -i

See the Installation Guide for detailed instructions.

What are the system requirements?

  • OS: macOS, Linux, or Windows (PowerShell)
  • Python: 3.11 or higher
  • Package Manager: uv (recommended) or pip
  • AI Provider: At least one API key (OpenAI, Anthropic, Google, Cerebras, or local model)

See Requirements for details.

How do I get started quickly?

  1. Install: uvx code-puppy -i
  2. Add a model: /add_model
  3. Start coding!

See the Quick Start Tutorial for a 5-minute walkthrough.

Common Tasks

How do I switch between agents?

Use the /agent command:

/agent code-puppy          # Switch to default agent
/agent code-reviewer       # Switch to code reviewer
/agent                     # Show available agents

See Switch Agents Guide for more details.

How do I add a new AI model?

Use the interactive /add_model command:

/add_model

This opens a TUI browser for models.dev where you can browse 65+ providers and add models with one click.

See Add Models Guide for more details.

How do I install a skill?

  1. Download or clone the skill into ~/.code_puppy/skills/
  2. Refresh: /skills refresh
  3. The skill is now available!

See Install Skills Guide for detailed instructions.

How do I create a custom agent?

You can create JSON agents manually or use the Agent Creator:

/agent agent-creator       # Use the guided creation wizard

See Create Agents Guide for step-by-step instructions.

How do I set up multiple models for load balancing?

Create a round-robin configuration in ~/.code_puppy/extra_models.json:

{
  "model1": { ... },
  "model2": { ... },
  "my_round_robin": {
    "type": "round_robin",
    "models": ["model1", "model2"],
    "rotate_every": 5
  }
}

See Round-Robin Models Guide for a complete tutorial.

Troubleshooting

I'm getting "No models configured"

Cause: No AI provider API keys are set.

Fix:

  1. Set an environment variable for your provider: ```bash export OPENAI_API_KEY=sk-... ```
  2. Or use /add_model to add a model from models.dev

The agent isn't responding

Try these steps:

  1. Check your API key is valid
  2. Verify network connectivity
  3. Try switching models: /model <different-model>
  4. Check if the model supports tool calling (some models don't)

A command isn't working

Try these steps:

  1. Check the command syntax: /help
  2. Verify you're in the right context (some commands only work in interactive mode)
  3. Check for typos in the command name

I'm getting authentication errors

Cause: Your API key is invalid or expired.

Fix:

  1. Re-authenticate: /chatgpt-auth, /claude-code-auth, or /antigravity-auth
  2. Check your environment variables
  3. Verify the API key hasn't expired

Configuration

How do I change settings?

Use the /set command:

/set yolo_mode true        # Enable yolo mode
/set skills_enabled false  # Disable skills

See Configuration Reference for all available settings.

Where is my configuration stored?

Configuration is stored in ~/.code_puppy/puppy.cfg.

How do I reset to defaults?

Delete or rename the config file:

mv ~/.code_puppy/puppy.cfg ~/.code_puppy/puppy.cfg.backup

Advanced

How do I create hooks?

Create a .claude/settings.json file with hook configurations:

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash|agent_run_shell_command",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/my-hook.sh"
          }
        ]
      }
    ]
  }
}

See Create Hooks Guide for detailed instructions.

How does the multi-agent system work?

Code Puppy has 20+ built-in agents, each specialized for different tasks:

  • code-puppy: General-purpose coding
  • code-reviewer: Code quality reviews
  • security-auditor: Security analysis
  • python-programmer: Python-specific tasks
  • And many more!

See Concepts: Agents for a detailed explanation.

Can I run Code Puppy locally without external APIs?

Yes! Configure a local model server (like Ollama, vLLM, or SGLang) and point Code Puppy to it:

{
  "my-local-model": {
    "type": "custom_openai",
    "name": "local-model",
    "custom_endpoint": {
      "url": "http://localhost:11434/v1",
      "api_key": "ollama"
    }
  }
}

See Model Configuration Guide for details.

Privacy

Does Code Puppy collect my data?

No. Code Puppy has a zero-compromise privacy policy:

  • ❌ No telemetry or analytics
  • ❌ No prompt logging
  • ❌ No behavioral tracking
  • ❌ No third-party data sharing

Your code and conversations stay on your machine unless you choose an external AI provider.

See Concepts: Privacy for details.

How can I ensure maximum privacy?

  1. Use local models (Ollama, vLLM, etc.)
  2. Don't set external API keys
  3. Run Code Puppy in an isolated environment
