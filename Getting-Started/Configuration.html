Configuration

Overview

This guide shows you how to configure Code Puppy to work with your preferred models, agents, and settings. Configuration is persistent and stored in ~/.code_puppy/puppy.cfg.

Prerequisites

Before you begin, make sure you have:

  • Code Puppy installed
  • API keys set for your chosen AI provider(s)

Quick Configuration

View Current Configuration

Use the /show command to see your current settings:

/show

You'll see output like:

Set Configuration Values

Use the /set command to change settings:

/set <key> <value>

Or use the = syntax:

/set key=value

Configuration Options

Basic Settings

Key Description Default Example
puppy_name Your Code Puppy's name Puppy /set puppy_name Fido
owner_name Your name Master /set owner_name Alice
default_agent Default agent to use code_puppy /set default_agent planner

Model Settings

Key Description Default Example
model Current model name First in models.json /set model claude-3-5-sonnet
temperature Global temperature (0.0-1.0) Model default /set temperature 0.7
openai_reasoning_effort Reasoning effort for GPT-5 medium /set openai_reasoning_effort high
openai_verbosity Response verbosity medium /set openai_verbosity low
TIP

Reasoning effort options: minimal, low, medium, high, xhigh Verbosity options: low, medium, high

Context Management

Key Description Default Example
protected_token_count Recent tokens to preserve 20,000 /set protected_token_count 30000
compaction_threshold Context usage % to trigger compaction 0.8 /set compaction_threshold 0.9
compaction_strategy How to compact context summarization /set compaction_strategy truncation
message_limit Maximum messages in history Unlimited /set message_limit 100

Session Management

Key Description Default Example
auto_save_session Auto-save after each response true /set auto_save_session false
max_saved_sessions Maximum saved sessions 10 /set max_saved_sessions 20
resume_message_count Messages shown on /resume 10 /set resume_message_count 20

Agent Behavior

Key Description Default Example
yolo_mode Allow risky operations false /set yolo_mode true
allow_recursion Allow recursive agent calls true /set allow_recursion false
enable_pack_agents Enable pack agents false /set enable_pack_agents true
enable_universal_constructor Enable dynamic tool creation true /set enable_universal_constructor false
enable_streaming Enable streaming responses true /set enable_streaming false
enable_dbos Enable durable execution true /set enable_dbos false
WARNING

DBOS changes require restart: When you change enable_dbos, you must restart Code Puppy for the change to take effect.

Keyboard Shortcuts

Key Description Default Example
cancel_agent_key Key to cancel agent tasks ctrl+c /set cancel_agent_key ctrl+k
WARNING

Cancel key changes require restart: When you change cancel_agent_key, you must restart Code Puppy for the change to take effect. Valid options: ctrl+c, ctrl+k, ctrl+q

Appearance

Key Description Default Example
banner_color_{name} Custom banner colors See defaults /set banner_color_primary #ff00ff

MCP & HTTP

Key Description Default Example
http2 Use HTTP/2 for requests true /set http2 false
diff_context_lines Lines of context in diffs 3 /set diff_context_lines 5

Environment Variables

You can also configure Code Puppy using environment variables:

API Keys

# OpenAI
export OPENAI_API_KEY=sk-...

# Anthropic
export ANTHROPIC_API_KEY=sk-ant-...

# Google
export GEMINI_API_KEY=AIza...

# Cerebras
export CEREBRAS_API_KEY=csk-...

DBOS Configuration

# DBOS conductor key (for DBOS Management Console)
export DBOS_CONDUCTOR_KEY=your_conductor_key

# DBOS logging level
export DBOS_LOG_LEVEL=INFO

# Custom database URL
export DBOS_SYSTEM_DATABASE_URL=postgresql://user:pass@localhost:5432/db

# DBOS app version (for workflow recovery)
export DBOS_APP_VERSION=1.0.0
TIP

Add environment variables to your shell profile (~/.bashrc, ~/.zshrc, or ~/.config/fish/config.fish) to persist them across sessions.

Model Configuration

Adding Models from models.dev

The easiest way to add models is using the /add_model command:

/add_model

This opens an interactive menu where you can:

  1. Browse 65+ AI providers
  2. Preview model details (pricing, context length, capabilities)
  3. Add models with one click

Manual Model Configuration

You can also add models manually to models.json (located in ~/.code_puppy/). See the AddModels guide for details.

Agent Configuration

Available Agents

Code Puppy comes with many built-in agents:

  • code_puppy - General purpose coding agent
  • planner - Task planning and decomposition
  • code_reviewer - Code review and feedback
  • security_auditor - Security analysis
  • qa_expert - Testing and quality assurance
  • And many more specialized agents

Switching Agents

Use the /agent command to switch agents:

/agent

Or specify an agent directly:

/agent code_reviewer

Pinning Models to Agents

You can pin specific models to agents using /pin_model:

/pin_model code_reviewer claude-3-5-sonnet

To unpin:

/pin_model code_reviewer (unpin)

MCP Server Configuration

MCP (Model Context Protocol) servers extend Code Puppy's capabilities. Configure them using:

/mcp

See the ManageMCP guide for details.

Hooks Configuration

Hooks let you customize Code Puppy's behavior. See the CreateHooks guide for details.

Verify Configuration

After making changes, verify they took effect:

/show

Common Issues

Problem Solution
Changes not taking effect Some settings require /reload or restart
Invalid config key Use /show to see available keys
API key not working Check environment variable is set correctly
Model not found Add it using /add_model or check models.json
