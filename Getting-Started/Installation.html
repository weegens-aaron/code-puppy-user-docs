Installation

Overview

This guide walks you through installing Code Puppy on your system. Code Puppy is designed to work on macOS, Linux, and Windows.

Prerequisites

Before you begin, make sure you have:

  • Python 3.11 or higher installed
  • UV package manager (recommended) or pip
  • API keys for the AI providers you want to use (see Configuration)

Step 1: Install UV (Recommended)

UV is the fastest way to run Code Puppy. It's a Python package installer and manager.

macOS / Linux

curl -LsSf https://astral.sh/uv/install.sh | sh

Windows (PowerShell as Admin)

powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
TIP

After installing UV, restart your terminal or run source ~/.bashrc (macOS/Linux) to make the uv command available.

Step 2: Run Code Puppy

The easiest way to use Code Puppy is with uvx, which runs the tool without requiring a global installation.

Quick Start

uvx code-puppy -i

This launches Code Puppy in interactive mode where you can start giving it tasks.

Verify Installation

uvx code-puppy --version

You should see something like:

code-puppy, version X.Y.Z

Step 3: Configure API Keys

Code Puppy supports multiple AI providers. You'll need API keys for the models you want to use.

Environment Variables

Set your API keys as environment variables:

# OpenAI (GPT models)
export OPENAI_API_KEY=sk-...

# Anthropic (Claude models)
export ANTHROPIC_API_KEY=sk-ant-...

# Google (Gemini models)
export GEMINI_API_KEY=AIza...

# Cerebras (fast inference)
export CEREBRAS_API_KEY=csk-...
TIP

Add these to your shell profile (~/.bashrc, ~/.zshrc, or ~/.config/fish/config.fish) to persist them across sessions.

Windows PowerShell

$env:OPENAI_API_KEY = "sk-..."
$env:ANTHROPIC_API_KEY = "sk-ant-..."

Alternative: Global Installation

If you prefer to install Code Puppy globally (so you can run code-puppy without uvx):

uv pip install --global code-puppy
code-puppy -i

Common Issues

Problem Solution
uvx: command not found Install UV first (Step 1)
No module named 'code_puppy' Make sure you're using uvx code-puppy, not uvx code_puppy
API key not found Set the appropriate environment variable for your chosen provider
Slow startup This is normal for the first run as dependencies are resolved
