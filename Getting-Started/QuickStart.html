Tutorial: Your First Code Task

What You'll Achieve

By the end of this tutorial, you'll have completed your first coding task with Code Puppy and learned the basic interaction patterns you'll use every day.

Time needed: ~10 minutes

You'll need:

  • Code Puppy installed and configured
  • An API key for any supported AI provider (OpenAI, Anthropic, Cerebras, Google, etc.)

You should already know:

  • Installation - How to install Code Puppy
  • Configuration - How to set up API keys

The Scenario

You're starting a new project and want to create a simple Python utility: a command-line todo list manager. This is a perfect first task because it:

  • Creates a new file from scratch
  • Uses basic shell commands
  • Shows how Code Puppy explains its work
  • Demonstrates the interactive nature of the tool

Step 1: Launch Code Puppy

Open your terminal and run:

uvx code-puppy -i

What happened: Code Puppy starts up, loads your configuration, and displays a welcome banner. You'll see something like:

🐶 Code Puppy vX.Y.Z
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

👋 Hello! I'm Puppy, your sassy code assistant.
   Ready to plow through some code with 1024 puppies?

Current agent: Code Puppy 🐶
Model: gpt-5
Type /help for commands or just start typing!
TIP

The -i flag launches interactive mode. You can also run uvx code-puppy without it, but -i gives you the full TUI experience.

Step 2: Give Your First Task

Type your request directly into the prompt:

Create a simple Python todo list manager as a command-line tool. It should support adding, listing, and completing tasks. Save it as todo.py

Press Enter.

What happened: Code Puppy receives your request, analyzes what needs to be done, and starts working. You'll see:

  1. Thinking indicator - A spinner showing Code Puppy is processing
  2. Reasoning - Code Puppy explains its approach
  3. Action - It creates the file using its file tools

Step 3: Watch Code Puppy Work

As Code Puppy works, you'll see streaming output like:

🤔 Analyzing requirements...

I'll create a todo list manager with:
- A Todo class to represent individual tasks
- A TodoList class to manage the collection
- CLI commands for add, list, and complete
- JSON file persistence to save tasks between runs

📝 Creating todo.py...

Then Code Puppy will show you the code it's creating:

#!/usr/bin/env python3
"""Simple command-line todo list manager."""

import json
from dataclasses import dataclass, asdict
from datetime import datetime
from pathlib import Path

@dataclass
class Todo:
    id: int
    text: str
    completed: bool = False
    created_at: str = ""
    
    def __post_init__(self):
        if not self.created_at:
            self.created_at = datetime.now().isoformat()

class TodoList:
    def __init__(self, storage_file: str = "todos.json"):
        self.storage_file = Path(storage_file)
        self.todos: list[Todo] = []
        self._load()
    
    def _load(self):
        if self.storage_file.exists():
            data = json.loads(self.storage_file.read_text())
            self.todos = [Todo(**t) for t in data]
    
    def _save(self):
        data = [asdict(t) for t in self.todos]
        self.storage_file.write_text(json.dumps(data, indent=2))
    
    def add(self, text: str) -> Todo:
        todo_id = max((t.id for t in self.todos), default=0) + 1
        todo = Todo(id=todo_id, text=text)
        self.todos.append(todo)
        self._save()
        return todo
    
    def list_all(self) -> list[Todo]:
        return self.todos
    
    def complete(self, todo_id: int) -> Todo | None:
        for todo in self.todos:
            if todo.id == todo_id:
                todo.completed = True
                self._save()
                return todo
        return None

def main():
    import sys
    todos = TodoList()
    
    if len(sys.argv) < 2:
        print("Usage: python todo.py <command> [args]")
        print("Commands: add <text>, list, complete <id>")
        return
    
    command = sys.argv[1]
    
    if command == "add":
        text = " ".join(sys.argv[2:])
        todo = todos.add(text)
        print(f"✓ Added task #{todo.id}: {todo.text}")
    
    elif command == "list":
        if not todos.todos:
            print("No tasks yet!")
        else:
            for todo in todos.todos:
                status = "✓" if todo.completed else "○"
                print(f"{status} #{todo.id}: {todo.text}")
    
    elif command == "complete":
        todo_id = int(sys.argv[2])
        todo = todos.complete(todo_id)
        if todo:
            print(f"✓ Completed task #{todo.id}")
        else:
            print(f"✗ Task #{todo_id} not found")

if __name__ == "__main__":
    main()
NOTE

Code Puppy doesn't just dump code - it explains what it's doing and why. This helps you understand and learn from the process.

Step 4: Test the Result

Code Puppy will typically test what it creates. You'll see:

🧪 Testing the todo list manager...

$ python todo.py add "Buy groceries"
✓ Added task #1: Buy groceries

$ python todo.py add "Walk the dog"
✓ Added task #2: Walk the dog

$ python todo.py list
○ #1: Buy groceries
○ #2: Walk the dog

$ python todo.py complete 1
✓ Completed task #1

$ python todo.py list
✓ #1: Buy groceries
○ #2: Walk the dog

What happened: Code Puppy ran shell commands to verify the code works correctly. It shows you the commands it ran and their output.

Step 5: Review and Iterate

Code Puppy will present the result and ask if you want changes:

✨ Done! I've created todo.py with a fully functional todo list manager.

Features:
✓ Add tasks with descriptions
✓ List all tasks with completion status
✓ Mark tasks as complete
✓ Persistent storage in todos.json

Want me to add anything? Maybe due dates, priorities, or categories?

You can now:

  • Accept - Say "Looks good!" to finish
  • Request changes - Say "Add due dates and priorities"
  • Ask questions - Say "How does the JSON persistence work?"

Step 6: Try a Follow-up Task

Let's add a feature:

Add due dates and priorities. Tasks should sort by priority (high first) then due date.

What happened: Code Puppy reads the existing file, understands the changes needed, and updates it while preserving existing functionality. This is called incremental development - Code Puppy builds on what already exists.

What You Learned

  • Natural language tasks - Just describe what you want in plain English
  • Streaming feedback - You see Code Puppy's thinking in real-time
  • Automatic testing - Code Puppy verifies its work
  • Incremental changes - You can refine and add features iteratively
  • Explanations - Code Puppy tells you what it's doing and why

Common First-Time Patterns

Asking Questions

How does the JSON persistence work?

Code Puppy will explain the code it just created.

Requesting Changes

Add error handling for invalid task IDs

Code Puppy will update the code with better error handling.

Exploring Options

What other features could I add?

Code Puppy will suggest enhancements based on common todo app patterns.

Troubleshooting

Problem Solution
Code Puppy doesn't respond Check your API key is set correctly
Slow response time First run takes longer as dependencies load
Code has errors Tell Code Puppy "There's an error: [paste error]"
Task takes too long Press Ctrl+C to cancel and try a simpler request

Tips for Success

TIP

Be specific but not prescriptive - Tell Code Puppy what you want, not exactly how to implement it. Let it use its expertise.

TIP

Iterate in small steps - It's better to ask for one feature at a time than to request a massive rewrite.

TIP

Ask questions - If you don't understand something Code Puppy did, just ask! It's a great learning tool.

TIP

Use agents for specialized tasks - Try /agent code_reviewer before accepting major changes for a code review.
