How to Add Models from models.dev

What You'll Learn

By the end of this guide, you'll be able to browse and add new AI models from the models.dev catalog to expand your available model options in Code Puppy.

Prerequisites

  • Code Puppy installed and running
  • Basic familiarity with the /add_model command
  • API credentials for the provider you want to add (you'll be prompted for these)

Quick Version

# Launch the model browser
/add_model

# Navigate with arrow keys, press Enter to add a model
# Follow credential prompts if needed

Detailed Steps

1. Launch the Model Browser

Type the following command in the Code Puppy terminal:

/add_model

You'll see a split-panel interface:

  • Left panel: List of AI providers (OpenAI, Anthropic, Groq, etc.)
  • Right panel: Details about the selected provider or model
TIP

The browser shows 1500+ models from the live models.dev database. It fetches fresh data on startup, with a bundled fallback if the API is unavailable.

2. Browse Providers

Use the ↑/↓ arrow keys to navigate through providers. Each provider shows:

  • Name and model count
  • ⚠️ icon if the provider is unsupported (requires special authentication)

Press → or Enter to view models for a provider.

Example:

 > Groq (45 models)
   Mistral (28 models)
   TogetherAI (120 models)

3. Browse Models for a Provider

Once you've selected a provider, you'll see its available models. Each model displays capability icons:

  • 👁 = Vision support
  • 🔧 = Tool calling support (important for coding tasks!)
  • 🧠 = Reasoning capabilities

Example:

 > 🔧🧠 claude-3-7-sonnet-20250219
   🔧 claude-3-5-sonnet-20241022
   👁🔧 gemini-1-5-pro-latest
WARNING

Models without tool calling (no 🔧 icon) will be very limited for coding tasks. They can't use file operations, shell commands, or other Code Puppy tools.

4. Review Model Details

The right panel shows comprehensive information:

  • Capabilities: Vision, tool calling, reasoning, temperature control
  • Pricing: Input/output token costs (if available)
  • Limits: Context window size, max output tokens
  • Metadata: Model ID, release date, open weights status

Use this to compare models before adding.

5. Add the Model

Press Enter on a model to add it to your configuration.

If credentials are missing, you'll be prompted:

🔑 Groq requires the following credentials:

  💡 Get your API key from https://console.groq.com/keys

  Enter GROQ_API_KEY (or press Enter to skip):

You can:

  • Enter the API key (saved to your config)
  • Press Enter to skip (set later with /set GROQ_API_KEY=<value>)

If all credentials are already set, you'll see:

✅ All required credentials for Groq are already set!

6. Add a Custom Model (Optional)

At the bottom of each provider's model list, you'll see:

 > ✨ Custom model...

Select this to add a model not listed in models.dev:

  1. Enter the exact model ID (e.g., gpt-4-turbo-preview)
  2. Enter the context window size (e.g., 128000 or 128k)
  3. Provide credentials if needed

Use cases for custom models:

  • Newly released models not yet in the catalog
  • Fine-tuned models
  • Preview/beta models
  • Custom deployments

Example: Adding Groq's Llama 3.1

Let's walk through a realistic example:

Step 1: Launch the browser

/add_model

Step 2: Navigate to Groq

 > Groq (45 models)

Step 3: Select a model

 > 🔧 llama-3.1-70b-versatile

Step 4: Review details

MODEL DETAILS

  Groq - llama-3.1-70b-versatile

  Capabilities:
    ✓ Vision
    ✓ Tool Calling
    ✓ Temperature

  Pricing:
    Input: $0.00000059/token
    Output: $0.00000099/token

  Limits:
    Context: 131072 tokens

Step 5: Add the model

✅ Added groq-llama-3.1-70b-versatile to extra_models.json

Step 6: Set credentials (if prompted)

Enter GROQ_API_KEY: gsk_xxxxxxxxxxxxxxxxxx
✅ Saved GROQ_API_KEY to config

Result: The model is now available! You can select it with /switch or /model.

Options & Variations

Option Description Example
Custom endpoint For providers with OpenAI-compatible APIs Uses custom_openai type automatically
Context size Override default context window Enter 32k or 32000 for custom models
Skip credentials Add model config now, set API key later Press Enter when prompted, use /set command

Troubleshooting

Problem Cause Solution
Provider shows ⚠️ Requires special authentication (AWS SigV4, GCP, etc.) Use a different provider or set up custom credentials manually
Model has no 🔧 icon Model doesn't support tool calling Choose a different model for coding tasks
Credentials prompt fails Terminal input issue on Windows Set env vars manually: set GROQ_API_KEY=xxx
Model not in catalog Newly released or custom model Use "Custom model..." option at bottom of list
extra_models.json error File format issue Check that it's a valid JSON dictionary, not a list

What Happens Behind the Scenes

When you add a model:

  1. Code Puppy creates an entry in extra_models.json
  2. The key is {provider-id}-{model-id} (e.g., groq-llama-3.1-70b)
  3. The value includes type, name, custom endpoint (if needed), and context length
  4. The model becomes immediately available for selection

**Example entry in extra_models.json:**

{
  "groq-llama-3.1-70b-versatile": {
    "type": "custom_openai",
    "name": "llama-3.1-70b-versatile",
    "custom_endpoint": {
      "url": "https://api.groq.com/openai/v1",
      "api_key": "$GROQ_API_KEY"
    },
    "context_length": 131072,
    "supported_settings": ["temperature", "seed", "top_p"]
  }
}
