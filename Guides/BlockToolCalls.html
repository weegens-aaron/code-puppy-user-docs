How to Block Tool Calls

What You'll Learn

By the end of this guide, you'll be able to create a plugin that intercepts and blocks specific tool calls before they execute, giving you fine-grained control over what actions the AI agent can perform.

Prerequisites

  • Familiarity with Code Puppy's plugin system
  • Basic understanding of Python async/await
  • Completed Create Hooks guide (recommended)

Quick Version

Create a plugin with a pre_tool_call callback that returns {"blocked": True, "error_message": "reason"}:

from code_puppy.callbacks import register_callback

async def block_dangerous_tools(tool_name: str, tool_args: dict, context=None):
    if tool_name == "delete_file":
        return {
            "blocked": True,
            "error_message": "🚫 File deletion is not allowed"
        }
    return None  # Allow other tools

register_callback("pre_tool_call", block_dangerous_tools)

Detailed Steps

1. Create Your Plugin Directory

Create a new plugin folder under code_puppy/plugins/ (builtin) or ~/.code_puppy/plugins/ (user):

mkdir -p ~/.code_puppy/plugins/block_tool_calls

2. Create the Callback Registration File

Create register_callbacks.py in your plugin directory:

"""Block specific tool calls before they execute."""

from typing import Any, Dict

from code_puppy.callbacks import register_callback


async def tool_blocker_callback(
    tool_name: str,
    tool_args: dict,
    context: Any = None
) -> Dict[str, Any] | None:
    """Intercept tool calls and block specific ones.

    Args:
        tool_name: Name of the tool being called
        tool_args: Arguments being passed to the tool
        context: Optional context data

    Returns:
        None to allow the tool call
        Dict with "blocked": True to block it
    """
    # Example: Block all file deletions
    if tool_name == "delete_file":
        return {
            "blocked": True,
            "error_message": "🚫 File deletion is blocked by policy"
        }

    # Example: Block shell commands with dangerous patterns
    if tool_name == "run_shell_command":
        command = tool_args.get("command", "")
        dangerous_patterns = ["rm -rf", "DROP TABLE", "chmod 777"]
        for pattern in dangerous_patterns:
            if pattern in command:
                return {
                    "blocked": True,
                    "error_message": f"🚫 Command blocked: contains '{pattern}'"
                }

    # Allow all other tools
    return None


# Register the callback
register_callback("pre_tool_call", tool_blocker_callback)

3. Understand the Blocking Mechanism

When a callback returns a dict with "blocked": True, Code Puppy:

  1. Cancels the tool execution - The tool never runs
  2. Shows a warning - User sees: 🚫 Hook blocked this tool call: {reason}
  3. Returns an error message - The agent receives: ERROR: The hook policy prevented this tool from running
  4. Allows agent recovery - The agent can acknowledge the block and try a different approach

4. Access Tool Arguments

The tool_args dict contains all parameters passed to the tool. Common tools and their arguments:

Tool Key Arguments
edit_file file_path, replacements
create_file file_path, content, overwrite
delete_file file_path
run_shell_command command, cwd, timeout
read_file file_path, start_line, num_lines

Example: Block edits to specific files:

async def protect_config_files(tool_name: str, tool_args: dict, context=None):
    protected = ["config.json", ".env", "secrets.yaml"]

    if tool_name in ("edit_file", "create_file", "replace_in_file"):
        file_path = tool_args.get("file_path", "")
        if any(file_path.endswith(p) for p in protected):
            return {
                "blocked": True,
                "error_message": f"🚫 Protected file: {file_path}"
            }

    return None

5. Implement Conditional Blocking

You can make blocking decisions based on context, user settings, or external conditions:

Example: Time-based restrictions:

import datetime

async def block_after_hours(tool_name: str, tool_args: dict, context=None):
    # Block destructive operations after 6 PM
    if datetime.datetime.now().hour >= 18:
        if tool_name in ("delete_file", "delete_snippet"):
            return {
                "blocked": True,
                "error_message": "🚫 Destructive operations blocked after business hours"
            }
    return None

Example: Environment-based restrictions:

import os

async def block_in_production(tool_name: str, tool_args: dict, context=None):
    if os.environ.get("ENV") == "production":
        if tool_name == "run_shell_command":
            command = tool_args.get("command", "")
            if "restart" in command or "deploy" in command:
                return {
                    "blocked": True,
                    "error_message": "🚫 Deployment commands blocked in production"
                }
    return None

Example: Complete Safety Policy Plugin

Here's a comprehensive example that implements a safety policy:

"""Safety policy plugin - blocks dangerous operations by default."""

import os
from typing import Any, Dict

from code_puppy.callbacks import register_callback


def is_destructive_tool(tool_name: str) -> bool:
    """Check if a tool can make destructive changes."""
    return tool_name in (
        "delete_file",
        "delete_snippet",
        "replace_in_file",
        "run_shell_command",
    )


def is_protected_path(file_path: str) -> bool:
    """Check if a file path is protected."""
    protected_patterns = [
        ".env",
        "secrets.",
        "config.prod.",
        "/etc/",
        "/var/",
    ]
    return any(p in file_path for p in protected_patterns)


async def safety_policy_callback(
    tool_name: str,
    tool_args: dict,
    context: Any = None
) -> Dict[str, Any] | None:
    """Enforce safety policy on tool calls."""

    # Always block edits to protected paths
    if tool_name in ("edit_file", "create_file", "replace_in_file"):
        file_path = tool_args.get("file_path", "")
        if is_protected_path(file_path):
            return {
                "blocked": True,
                "error_message": f"🚫 Protected path: {file_path}"
            }

    # Block dangerous shell commands
    if tool_name == "run_shell_command":
        command = tool_args.get("command", "")

        # Block destructive commands
        dangerous = [
            "rm -rf /",
            "rm -rf *",
            "mkfs",
            "dd if=",
            "DROP TABLE",
            "TRUNCATE",
        ]
        for pattern in dangerous:
            if pattern in command:
                return {
                    "blocked": True,
                    "error_message": f"🚫 Dangerous command blocked: {pattern}"
                }

    # Require explicit approval for all destructive tools
    if is_destructive_tool(tool_name):
        # In a real plugin, you might check a config setting
        # or prompt the user here
        pass  # Allow for now, but log the attempt

    return None


register_callback("pre_tool_call", safety_policy_callback)

Options & Variations

Pattern Description Example
Block by tool name Block all calls to a specific tool if tool_name == "delete_file"
Block by argument value Block based on specific arguments if "rm -rf" in tool_args["command"]
Block by file path Protect specific files/directories if file_path.endswith(".env")
Block by time/context Conditional blocking if datetime.now().hour > 18
Allowlist mode Only allow specific tools if tool_name not in allowed_tools

Troubleshooting

Problem Cause Solution
Tool still executes after blocking Callback returns wrong format Ensure you return {"blocked": True, ...} not just True
Agent crashes on block Error in callback Wrap callback in try/except, return None on error
Multiple blocks show confusing messages Multiple plugins blocking Only one block message is shown (first one wins)
Callback not being called Plugin not loaded Check plugin directory is in ~/.code_puppy/plugins/
