How to Use Slash Commands

What You'll Learn

By the end of this guide, you'll be able to use all of Code Puppy's slash commands to control agents, manage sessions, configure settings, and interact with MCP servers.

Prerequisites

  • Code Puppy installed and running
  • Basic familiarity with the interactive prompt

Quick Version

Type a command starting with / and press Enter:

/help          # Show all available commands
/model         # Switch to a different model
/agent         # Switch to a different agent
/set yolo_mode true   # Enable YOLO mode

Detailed Steps

1. Understanding Command Syntax

Slash commands always start with / followed by the command name:

/command_name
/command_name argument1 argument2
/command_name --flag value

Aliases: Many commands have shorter aliases:

  • /model or /m - Set active model
  • /agent or /a - Switch agents
  • /session or /s - Manage sessions
  • /help or /h - Show help

2. Getting Help

Show all commands:

/help

This displays:

  • Built-in Commands (all core functionality)
  • Custom Commands (from plugins)
  • Shell Pass-through (run shell commands with !)

3. Core Commands

Navigation & Session

Command Description Example
/cd <dir> Change working directory /cd ~/projects/my-app
/tools Show available tools /tools
/motd Show message of the day /motd
/tutorial Run interactive tutorial /tutorial
/exit or /quit Exit interactive mode /exit

Agent & Model Control

Command Description Example
/agent or /a Show agent picker /agent
/agent <name> Switch to specific agent /agent code_reviewer
/model or /m Show model picker /model
/model <name> Set active model /model claude-opus-4-6
/add_model Browse models.dev catalog /add_model
/model_settings or /ms Configure per-model settings /model_settings

Configuration

Command Description Example
/show Display current configuration /show
/set <key> <value> Set config value /set yolo_mode true
/reasoning <level> Set reasoning effort (GPT-5) /reasoning high
/verbosity <level> Set verbosity (GPT-5) /verbosity low
/pin_model <agent> <model> Pin model to agent /pin_model code_reviewer claude-opus-4-6
/unpin <agent> Unpin model from agent /unpin code_reviewer
/diff Configure diff colors /diff
/colors Configure banner colors /colors

Session Management

Command Description Example
/session or /s Show current session ID /session
/session new Create new session /session new
/compact Compact message history /compact
/truncate <N> Keep only N recent messages /truncate 10
/autosave_load or /resume Load autosaved session /autosave_load
/dump_context <name> Save context to file /dump_context my-work
/load_context <name> Load context from file /load_context my-work

4. Shell Pass-through

Run shell commands directly with !:

!git status
!ls -la
!pwd

5. Configuration Examples

Enable YOLO mode (unrestricted execution):

/set yolo_mode true

Set reasoning effort for GPT-5:

/reasoning high

Set per-model temperature:

/model_settings
# Then select your model and adjust temperature

Pin a model to a specific agent:

/pin_model code_reviewer claude-opus-4-6

Compact history to save tokens:

/compact

Example Workflow

Scenario: You want to switch to a code reviewer agent with a specific model and start working.

# 1. Check current configuration
/show

# 2. Switch to code reviewer agent
/agent code_reviewer

# 3. Set the model you want to use
/model claude-opus-4-6

# 4. Verify the switch
/show

# 5. Start your task
# (Type your prompt and press Enter)

Options & Variations

Config Keys for /set

Key Values Description
yolo_mode true / false Enable unrestricted execution
auto_save_session true / false Auto-save after each response
enable_dbos true / false Enable DBOS (requires restart)
cancel_agent_key ctrl+c, ctrl+k, ctrl+q Key to cancel agent
protected_token_count Number Recent tokens to preserve
compaction_threshold Percentage (0.0-1.0) When to compact
compaction_strategy summarization / truncation How to compact
resume_message_count Number Messages shown on resume

Reasoning Effort Levels

  • minimal - Fastest, least reasoning
  • low - Quick reasoning
  • medium - Balanced (default)
  • high - Thorough reasoning
  • xhigh - Maximum reasoning

Verbosity Levels

  • low - More concise responses
  • medium - Balanced (default)
  • high - More detailed responses

Troubleshooting

Problem Cause Solution
Unknown command: /xyz Command doesn't exist Type /help to see available commands
Command not working Agent may need reload Run /show to verify, or switch agents
Model settings not applied Agent not reloaded The agent auto-reloads, or use /agent to force refresh
/mcp shows no servers No servers registered Use /mcp search to find servers, then /mcp install
Skills not showing Skills system disabled Use /skills toggle to enable
