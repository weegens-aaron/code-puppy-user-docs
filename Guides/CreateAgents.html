How to Create Custom Agents

What You'll Learn

By the end of this guide, you'll be able to create your own custom agents with specialized expertise, tool access, and system prompts tailored to your specific needs.

Prerequisites

  • Code Puppy installed and running
  • Familiarity with slash commands for switching agents
  • Basic understanding of Python (for Python-based agents)

Quick Version

For JSON agents (simplest):

mkdir -p ~/.code_puppy/agents/my-agent
cat > ~/.code_puppy/agents/my-agent/agent.json << 'EOF'
{
  "name": "my-agent",
  "display_name": "My Custom Agent",
  "description": "Does something specific",
  "system_prompt": "You are a helpful assistant...",
  "tools": ["list_files", "read_file", "create_file"]
}
EOF

For Python agents (advanced):

# Create agent file
cat > code_puppy/agents/agent_my_agent.py << 'EOF'
from .base_agent import BaseAgent

class MyAgent(BaseAgent):
    @property
    def name(self): return "my-agent"
    
    @property
    def display_name(self): return "My Agent 🐾"
    
    @property
    def description(self): return "Custom agent description"
    
    def get_system_prompt(self): return "Your system prompt here"
    
    def get_available_tools(self): return ["list_files", "read_file"]
EOF

# Restart Code Puppy

Detailed Steps

1. Choose Your Agent Type

Code Puppy supports two types of custom agents:

Type Best For Complexity Location
JSON Agent Simple customizations, quick prototyping Easy ~/.code_puppy/agents/
Python Agent Complex logic, dynamic behavior Advanced code_puppy/agents/
TIP

Start with JSON agents for simplicity. Use Python agents when you need dynamic behavior or complex tool logic.

2. Creating a JSON Agent

JSON agents are the simplest way to create a custom agent.

Step 2.1: Create the Agent Directory

mkdir -p ~/.code_puppy/agents/my-python-helper

Step 2.2: Create the agent.json File

Create ~/.code_puppy/agents/my-python-helper/agent.json:

{
  "name": "my-python-helper",
  "display_name": "Python Helper 🐍",
  "description": "Specialized Python code reviewer and refactoring assistant",
  "system_prompt": "You are an expert Python developer. Your job is to review code for Pythonic patterns, type safety, and performance. Always suggest improvements with concrete examples.",
  "tools": [
    "list_files",
    "read_file",
    "grep",
    "replace_in_file",
    "agent_run_shell_command"
  ],
  "user_prompt": "What Python code would you like me to review?"
}

Step 2.3: Verify the Agent

Restart Code Puppy and check if your agent appears:

/code_puppy
# Then use /agents to list available agents

You should see "Python Helper 🐍" in the agent list.

3. Creating a Python Agent

Python agents give you full control over agent behavior.

Step 3.1: Create the Agent File

Create a new file: code_puppy/agents/agent_my_specialist.py

"""My specialist agent for specific tasks."""

from .base_agent import BaseAgent


class MySpecialistAgent(BaseAgent):
    """Custom agent for specialized tasks."""

    @property
    def name(self) -> str:
        return "my-specialist"

    @property
    def display_name(self) -> str:
        return "My Specialist 🎯"

    @property
    def description(self) -> str:
        return "Specialized agent for [your specific use case]"

    def get_system_prompt(self) -> str:
        return """
You are a [your role] expert. Your mission is to [your mission].

Your expertise includes:

### Core Skills
- Skill 1: Description
- Skill 2: Description
- Skill 3: Description

### Best Practices
1. Always do X before Y
2. Never forget to Z
3. Prefer A over B when...

### Workflow
1. First, analyze the problem
2. Then, propose a solution
3. Finally, implement with tests

### Personality
- Be [trait 1]
- Be [trait 2]
- Use [tone/style]

Remember: [key principle]
"""

    def get_available_tools(self) -> list[str]:
        """List tools this agent needs."""
        return [
            "list_files",
            "read_file",
            "create_file",
            "replace_in_file",
            "agent_run_shell_command",
            # Add more tools as needed
        ]

    def get_user_prompt(self) -> str:
        """Optional: Custom user prompt."""
        return "What would you like me to help you with?"

Step 3.2: Follow the Required Interface

Every Python agent must implement these required properties and methods:

Member Type Description
name property Unique identifier (lowercase, hyphens)
display_name property Human-readable name with emoji
description property Brief description shown in agent list
get_system_prompt() method Returns the system prompt string
get_available_tools() method Returns list of tool names

Optional methods:

Member Type Description
get_user_prompt() method Custom user prompt
get_tools_config() method Tool configuration dict

Step 3.3: Name Your Agent File Correctly

Agent files must follow this naming pattern:

agent_<agent-name>.py

Examples:

  • agent_python_reviewer.py → agent name: python-reviewer
  • agent_security_auditor.py → agent name: security-auditor
  • agent_my_custom.py → agent name: my-custom
WARNING

The filename determines the agent's name property. Use lowercase with hyphens.

Step 3.4: Restart Code Puppy

Python agents are discovered at startup. Restart Code Puppy to load your new agent:

# Exit current session (Ctrl+C or /quit)
# Then restart
code_puppy

4. Testing Your Agent

Step 4.1: Switch to Your Agent

Use the /agents command to switch:

/agents
# Select your agent from the list

Or use the command directly:

/switch-agent my-specialist

Step 4.2: Verify Tool Access

Ask your agent to use its tools:

Can you list the files in the current directory?

You should see the agent use the list_files tool.

Step 4.3: Check the System Prompt

Your agent should respond according to its specialized instructions. If it doesn't, verify:

  1. The get_system_prompt() method returns the correct string
  2. The file is in the correct location
  3. Code Puppy was restarted after creating the agent

Example: Security Audit Agent

Here's a complete example of a security-focused agent:

"""Security audit agent for code reviews."""

from .base_agent import BaseAgent


class SecurityAuditorAgent(BaseAgent):
    """Specialized security code reviewer."""

    @property
    def name(self) -> str:
        return "security-auditor"

    @property
    def display_name(self) -> str:
        return "Security Auditor 🔒"

    @property
    def description(self) -> str:
        return "Expert security reviewer for vulnerabilities, OWASP compliance, and secure coding"

    def get_system_prompt(self) -> str:
        return """
You are a cybersecurity expert specializing in application security.

Your mission: Identify security vulnerabilities and recommend fixes.

## Security Domains

### Input Validation
- SQL injection prevention
- XSS (Cross-Site Scripting) mitigation
- Command injection protection
- Path traversal attacks

### Authentication & Authorization
- Weak password policies
- Session management flaws
- Broken access control
- Privilege escalation risks

### Data Protection
- Sensitive data exposure
- Insecure cryptographic storage
- Missing encryption in transit
- Weak cryptographic algorithms

### Configuration
- Security misconfiguration
- Default credentials
- Unnecessary services enabled
- Exposed sensitive data in configs

## Review Process

1. **Scan**: Identify all entry points and data flows
2. **Analyze**: Look for security anti-patterns
3. **Prioritize**: Rate findings by severity (Critical/High/Medium/Low)
4. **Remediate**: Provide concrete fix examples

## Output Format

For each vulnerability found:

[SEVERITY] Vulnerability Name

  • Location: file.py:line
  • Issue: Description of the problem
  • Risk: What could happen if exploited
  • Fix: Code example showing the fix

## Tone

- Be direct and factual
- Provide actionable recommendations
- Explain the "why" behind security rules
- Don't be alarmist, but don't downplay risks

Remember: Security is about enabling safe innovation, not blocking progress.
"""

    def get_available_tools(self) -> list[str]:
        return [
            "list_files",
            "read_file",
            "grep",
            "agent_run_shell_command",  # For running security scanners
            "create_file",  # For writing security reports
        ]

    def get_user_prompt(self) -> str:
        return "What codebase would you like me to audit for security vulnerabilities?"

Options & Variations

Dynamic Tool Selection

Python agents can implement logic in get_available_tools():

def get_available_tools(self) -> list[str]:
    tools = ["list_files", "read_file"]
    
    # Add browser tools if available
    if self._browser_available():
        tools.append("browser_navigate")
        tools.append("browser_screenshot")
    
    return tools

def _browser_available(self) -> bool:
    # Check if browser tools are configured
    from code_puppy.config import get_value
    return get_value("browser_enabled", False)

Custom Tool Configuration

Use get_tools_config() to customize tool behavior:

def get_tools_config(self) -> dict:
    return {
        "agent_run_shell_command": {
            "timeout": 300,  # 5 minute timeout
            "allow_dangerous": False,  # Block destructive commands
        }
    }

Multi-Model Support

Agents can work with different models:

from code_puppy.config import get_agent_pinned_model

def get_model_name(self) -> str:
    # Check if this agent has a pinned model
    pinned = get_agent_pinned_model(self.name)
    if pinned:
        return pinned
    # Otherwise use global default
    from code_puppy.config import get_global_model_name
    return get_global_model_name()

Troubleshooting

Problem Cause Solution
Agent doesn't appear in list File not in correct location Move to code_puppy/agents/ or ~/.code_puppy/agents/
Agent appears but fails to load Syntax error in Python file Check ~/.code_puppy/error_logs/ for details
Tools don't work Tool name misspelled Verify tool names match available tools
System prompt not applied Method not implemented Ensure get_system_prompt() is defined
JSON agent not loaded Invalid JSON Run `cat agent.json \ python -m json.tool` to validate

Next Steps

Now that you've created your custom agent:

  1. Test it: Switch to your agent and try some tasks
  2. Refine it: Adjust the system prompt based on results
  3. Share it: Consider open-sourcing your agent for others to use
  4. Extend it: Add skills to give your agent more capabilities
TIP

Keep your agents focused. A specialized agent that does one thing well is better than a generalist that does everything poorly.
