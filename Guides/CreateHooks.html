How to Create Hooks

What You'll Learn

By the end of this guide, you'll be able to create custom hooks to intercept and control agent tool calls, block dangerous commands, and extend Code Puppy's behavior without modifying core code.

Prerequisites

  • Basic understanding of Code Puppy's command-line interface
  • Familiarity with Python (for callback-based hooks) or shell scripting (for script-based hooks)
  • Completed Getting Started tutorial

Quick Version

Script-based hooks (Claude Code compatible):

// .claude/settings.json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "bash .claude/hooks/my-check.sh"
      }]
    }]
  }
}

Python callback hooks (Code Puppy native):

# ~/.code_puppy/plugins/my_plugin/register_callbacks.py
from code_puppy.callbacks import register_callback

def my_hook():
    print("Hook executed!")

register_callback("startup", my_hook)

Detailed Steps

Understanding Hook Types

Code Puppy supports two types of hooks:

Type Use Case Location
Script-based Block/modify tool calls, integrate with external tools .claude/settings.json + scripts
Python callbacks Extend Code Puppy behavior, add custom commands ~/.code_puppy/plugins/

Script-Based Hooks (PreToolUse / PostToolUse)

Step 1: Create Your Hook Configuration

Create .claude/settings.json in your project root:

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash|agent_run_shell_command",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/safety-check.sh",
            "timeout": 5000
          }
        ]
      }
    ]
  }
}
TIP

Use Bash|agent_run_shell_command to match both Claude Code and Code Puppy tool names.

Step 2: Create the Hook Script

Create .claude/hooks/safety-check.sh:

#!/bin/bash

# Read JSON from stdin
input=$(cat)

# Extract the command being run
command=$(echo "$input" | jq -r '.tool_input.command // empty')

# Check for dangerous commands
if echo "$command" | grep -qE '(^|\s)(rm -rf|sudo rm|mkfs)(\s|$)'; then
  echo "🛑 Blocked: Dangerous command detected" >&2
  echo "Command: $command" >&2
  exit 1  # Block the tool call
fi

# Allow the command
exit 0

Make it executable:

chmod +x .claude/hooks/safety-check.sh

Step 3: Test Your Hook

Run Code Puppy and try to execute a blocked command:

> /ask Run: rm -rf /important/files

You should see:

🛑 Blocked: Dangerous command detected
Command: rm -rf /important/files

Step 4: Access Hook Input Data

Your script receives JSON on stdin with this structure:

{
  "session_id": "codepuppy-session",
  "hook_event_name": "PreToolUse",
  "tool_name": "agent_run_shell_command",
  "tool_input": {
    "command": "echo hello"
  },
  "cwd": "/path/to/project",
  "permission_mode": "default"
}

Common jq queries:

# Get tool name
echo "$input" | jq -r '.tool_name'

# Get command argument
echo "$input" | jq -r '.tool_input.command'

# Get file path (for Edit/Read/Write tools)
echo "$input" | jq -r '.tool_input.path'

Step 5: Use Environment Variables

Hook scripts also have access to these environment variables:

Variable Description
CLAUDE_TOOL_INPUT JSON string of tool arguments
CLAUDE_TOOL_NAME Name of the tool being called
CLAUDE_HOOK_EVENT Event type (PreToolUse, PostToolUse)
CLAUDE_PROJECT_DIR Absolute path to project root
CLAUDE_FILE_PATH File path if tool targets a file

Example using environment variables:

#!/bin/bash
echo "Tool: $CLAUDE_TOOL_NAME"
echo "Project: $CLAUDE_PROJECT_DIR"

Python Callback Hooks (Code Puppy Native)

Step 1: Create a Plugin Directory

Create a plugin in your user plugins directory:

mkdir -p ~/.code_puppy/plugins/my_plugin

Step 2: Create the Callback Registration File

Create ~/.code_puppy/plugins/my_plugin/register_callbacks.py:

from code_puppy.callbacks import register_callback
from code_puppy.messaging import emit_info

def my_startup_hook():
    """Runs when Code Puppy starts."""
    emit_info("🐶 My plugin loaded successfully!")

def my_custom_command_handler(command: str, name: str):
    """Handle custom slash commands."""
    if name == "hello":
        emit_info("Hello! How can I help?")
        return True  # Indicate we handled this command
    return None  # Not handled

# Register the callbacks
register_callback("startup", my_startup_hook)
register_callback("custom_command", my_custom_command_handler)

Step 3: Restart Code Puppy

Your plugin will be auto-discovered on the next startup. You should see:

🐶 My plugin loaded successfully!

Step 4: Use Your Custom Command

Now you can use your custom command:

> /hello

Available Hook Types

Script-Based Hooks (.claude/settings.json)

Event When It Fires Can Block?
PreToolUse Before any tool call ✅ Yes (exit 1)
PostToolUse After tool completes ❌ No (observation only)
SessionStart When session begins ❌ No
Stop When agent finishes task ✅ Yes
SubagentStop When sub-agent finishes ✅ Yes

Python Callback Hooks

Callback When It Fires Typical Use
startup App boot Initialize resources, log startup
shutdown Graceful exit Cleanup, save state
run_shell_command Before shell exec Safety checks, logging
pre_tool_call Before any tool Inspect/modify tool args
post_tool_call After tool finishes Log results, metrics
custom_command Unknown /slash cmd Add custom commands
custom_command_help /help menu Add help entries
file_permission Before file op Access control, audit
agent_run_start Agent task begins Start background tasks
agent_run_end Agent task ends Stop background tasks
stream_event Response streaming Real-time monitoring

Example: Block Git Commands

Script-Based Version

.claude/settings.json:

{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash|agent_run_shell_command",
      "hooks": [{
        "type": "command",
        "command": "bash .claude/hooks/no-git.sh"
      }]
    }]
  }
}

.claude/hooks/no-git.sh:

#!/bin/bash
input=$(cat)
command=$(echo "$input" | jq -r '.tool_input.command // empty')

if echo "$command" | grep -qE '(^|\s)git(\s|$)'; then
  echo "🛑 Git commands are not allowed. Use the PR workflow." >&2
  exit 1
fi
exit 0

Example: Log All Tool Calls

Python Callback Version

~/.code_puppy/plugins/tool_logger/register_callbacks.py:

import json
from pathlib import Path
from code_puppy.callbacks import register_callback

LOG_FILE = Path.home() / ".code_puppy" / "tool_log.jsonl"

def log_tool_call(tool_name: str, tool_args: dict, context=None):
    """Log every tool call to a file."""
    entry = {
        "tool": tool_name,
        "args": tool_args,
        "context": str(context) if context else None
    }
    with open(LOG_FILE, "a") as f:
        f.write(json.dumps(entry) + "\n")

register_callback("pre_tool_call", log_tool_call)

Matcher Syntax

Basic Matchers

"Bash"                    # Exact match (Claude Code name)
"agent_run_shell_command" # Exact match (internal name)
"Bash|agent_run_shell"    # OR — matches either
"*"                       # Wildcard — matches everything

Compound Matchers

"Edit && .py"             # AND — Edit tool AND .py file
"Read || Write"           # OR — Read or Write tool

File Extension Matchers

"Edit && .py"   # Python files only
"Read && .md"   # Markdown files only
"Write && .txt" # Text files only

Exit Codes (Script-Based Hooks)

Code Meaning Effect
0 Allow Tool runs normally. Stdout shown in transcript.
1 Block Tool is prevented. Stderr shown as block reason.
2 Error feedback Stderr fed back to agent. Tool still runs.
WARNING

Always exit with an explicit code. Missing exit statements default to 0 (allow).

Troubleshooting

Hook Isn't Running

Check:

  1. Is the script executable? chmod +x .claude/hooks/my-hook.sh
  2. Is the matcher correct? Try "*" temporarily to test
  3. Check startup logs for: Hook engine ready - Total: N

Test standalone:

echo '{"tool_input":{"command":"echo test"}}' | bash .claude/hooks/my-hook.sh

JSON Parsing Fails

Problem: Using wrong field names

Fix: Use .tool_input.command (not .command) — args are nested under tool_input

Python Callback Not Loading

Check:

  1. Is the file named register_callbacks.py?
  2. Is it in a subdirectory of ~/.code_puppy/plugins/?
  3. Check logs for plugin loading errors
