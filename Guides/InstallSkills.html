How to Install Skills

What You'll Learn

By the end of this guide, you'll be able to browse, install, and manage agent skills that extend Code Puppy's capabilities.

Prerequisites

  • Code Puppy installed and running
  • Basic familiarity with the /skills command

Quick Version

# Launch the skills browser
/skills

# Install from remote catalog
/skills install

What Are Skills?

Skills are reusable, community-shared extensions that teach Code Puppy new capabilities. Each skill can include:

  • Scripts - Shell commands or Python scripts the agent can run
  • References - Documentation, guidelines, or context files
  • Metadata - Name, description, tags for discovery

When a skill is installed and enabled, Code Puppy automatically knows about it and can use it when appropriate.

Installing Skills from the Remote Catalog

Step 1: Launch the Skills Browser

Open Code Puppy and run:

/skills install

You'll see a split-panel interface:

  • Left panel: Skill categories
  • Right panel: Details for the selected item

Step 2: Browse Categories

Use the arrow keys to navigate through available categories:

Category Icon Description
Data 📊 Data analysis and processing
Finance 💰 Financial calculations and reporting
Legal ⚖️ Legal document review
Office 📄 Office productivity tasks
Product Management 📦 Product development workflows
Sales 💼 Sales and CRM tasks
Biology 🧬 Biological and scientific tasks

Press Enter on a category to see its skills.

Step 3: Select a Skill

Navigate to a skill you want to install. The right panel shows:

  • Description - What the skill does
  • ID - Unique identifier
  • Category & Tags - For organization
  • Contents - Scripts, references, file count
  • Download Size - How much to download

Skills already installed show a ✓ icon.

Step 4: Install

Press Enter on a skill to install it. You'll see:

Install skill 'data-csv-parser' (245 KB)? [y/N]

Type y and press Enter to confirm.

You should see:

✓ Successfully installed skill: data-csv-parser
Installed to: ~/.code_puppy/skills/data-csv-parser

Step 5: Verify Installation

Run /skills to see your installed skills. The skill should appear with a ✓ icon.

Managing Local Skills

Viewing All Skills

/skills list

This shows all discovered skills with their status:

🛠️ Skills (integration: enabled)
Found 3 skill(s):

  🟢 enabled data-csv-parser v1.0.0 by John Doe
      Parse and manipulate CSV files
      tags: data, csv, parsing

  🟢 enabled pdf-analyzer v2.1.0
      Extract text and metadata from PDFs
      tags: pdf, documents, analysis

  🔴 disabled legacy-tool v0.5.0
      Deprecated utility (disabled)

Enabling/Disabling Skills

In the /skills browser:

  • Navigate to a skill
  • Press Enter to toggle enabled/disabled

Or use commands:

/skills enable data-csv-parser
/skills disable legacy-tool

Adding Custom Skill Directories

By default, Code Puppy looks for skills in:

  • ~/.code_puppy/skills/
  • ./.code_puppy/skills/
  • ./skills/

To add a custom directory:

  1. Run /skills
  2. Press Ctrl+A
  3. Enter the directory path
  4. Press Enter to confirm

Example:

Directory path: /opt/shared-skills

Removing Skill Directories

  1. Run /skills
  2. Press Ctrl+D
  3. Enter the number of the directory to remove
  4. Confirm with y

Creating Your Own Skills

Basic Skill Structure

Create a directory with this structure:

my-skill/
├── SKILL.md          # Required: metadata and description
├── scripts/          # Optional: executable scripts
│   └── my_script.sh
└── references/       # Optional: documentation files
    └── guide.md

SKILL.md Format

---
name: my-skill
version: 1.0.0
author: Your Name
description: Does something useful
tags:
  - utility
  - helper
---

# My Skill

Detailed description of what this skill does.
Include usage examples and any prerequisites.

Example: A Simple Git Helper Skill

Create ~/.code_puppy/skills/git-helper/:

SKILL.md:

---
name: git-helper
version: 1.0.0
author: You
description: Common Git operations helper
tags:
  - git
  - version-control
---

# Git Helper

Provides common Git operations like status checks, branch management, and commit helpers.

scripts/git-status.sh:

#!/bin/bash
git status --short

Installing Your Local Skill

  1. Create the skill directory structure
  2. Run /skills to refresh
  3. Your skill should appear in the list
  4. Press Enter to enable it

Tips & Best Practices

TIP

Skill Discovery: Skills with complete SKILL.md metadata are easier to find and use. Always include name, description, and tags.

TIP

Disable Instead of Delete: If you're not sure about a skill, disable it rather than deleting. You can re-enable it anytime.

TIP

Custom Directories: Use custom skill directories to share skills across projects or with your team.

Troubleshooting

Problem Cause Solution
"No skills found" No skills installed Use /skills install to browse the catalog
Skill not appearing Missing SKILL.md Add a valid SKILL.md file to the skill directory
Installation fails Network issue Check your internet connection and try again
Skill shows as disabled Explicitly disabled Run /skills enable <skill-name>
