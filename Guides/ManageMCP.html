How to Manage MCP Servers

What You'll Learn

By the end of this guide, you'll be able to install, start, stop, and manage Model Context Protocol (MCP) servers to extend Code Puppy's capabilities.

Prerequisites

  • Code Puppy installed and running
  • Basic understanding of what MCP servers are (see Concepts: MCP Integration)

Quick Version

# List all servers
/mcp list

# Install a server from catalog
/mcp install <server_id>

# Start a server
/mcp start <server_name>

# Check status
/mcp status <server_name>

# Stop a server
/mcp stop <server_name>

# Remove a server
/mcp remove <server_name>

Detailed Steps

1. List Available Servers

First, see what MCP servers are currently registered:

/mcp list

You'll see a table showing:

  • Name: Server identifier
  • Type: stdio, sse, or streamable-http
  • State: running, stopped, starting, error
  • Enabled: Whether the server is active
  • Uptime: How long the server has been running
  • Status: Current health or error message

Example output:

🔌 MCP Server Status Dashboard
┌──────────────┬────────┬──────────┬───────────┬──────────┬────────────┐
│ Name         │ Type   │ State    │ Enabled   │ Uptime   │ Status     │
├──────────────┼────────┼──────────┼───────────┼──────────┼────────────┤
│ filesystem   │ STDIO  │ 🟢 running │ ✓        │ 2h 15m   │ OK         │
│ memory       │ STDIO  │ 🔴 stopped │ ✗        │ -        │ OK         │
└──────────────┴────────┴──────────┴───────────┴──────────┴────────────┘

📊 Summary: 1/2 servers running

2. Install a Server from Catalog

Code Puppy includes a catalog of pre-configured MCP servers:

/mcp install

This launches an interactive menu showing available servers. Browse with arrow keys and press Enter to install.

Install a specific server by ID:

/mcp install filesystem

During installation, you may be prompted for:

  • Custom name: How you want to refer to this server
  • Environment variables: API keys or configuration values
  • Command-line arguments: Server-specific options
TIP

Use /mcp install without arguments to browse the full catalog interactively.

3. Start a Server

Once installed, start the server:

/mcp start filesystem

For stdio servers (most common):

  • The server subprocess starts asynchronously in the background
  • You'll see: 🚀 Starting server: filesystem (subprocess starting in background)
  • The server is ready when its state shows running

For SSE/HTTP servers:

  • The server is enabled immediately
  • Connection happens on first use
NOTE

After starting, Code Puppy automatically reloads the agent to pick up the new server tools.

4. Check Server Status

Check the status of a specific server:

/mcp status filesystem

This shows detailed information:

  • Server ID and type
  • Current state (running/stopped/error)
  • Whether it's enabled
  • Uptime
  • Recent events
  • Any error messages

Example output:

🔌 filesystem Status
Server: filesystem
ID: filesystem
Type: STDIO
State: 🟢 running
Enabled: ✓ Yes
Process: ✓ Active (subprocess/connection running)
Uptime: 2h 15m
Recent Events: 12

5. Stop a Server

Stop a server when you're done using it:

/mcp stop filesystem

This:

  • Disables the server
  • Stops the subprocess (for stdio servers)
  • Reloads the agent to remove the server's tools

6. Remove a Server

Permanently remove a server:

/mcp remove filesystem

This:

  • Stops the server if running
  • Removes it from the configuration
  • Updates mcp_servers.json
WARNING

This cannot be undone. You'll need to reinstall the server if you want it back.

Example: Setting Up a Filesystem Server

Here's a complete workflow to add filesystem capabilities:

Step 1: Install the server

/mcp install filesystem

Step 2: Configure (if prompted)

  • Custom name: filesystem (or your preferred name)
  • No environment variables required for basic filesystem access

Step 3: Start the server

/mcp start filesystem

Step 4: Verify it's running

/mcp status filesystem

Step 5: Use the server Now you can ask Code Puppy to work with files:

"Read the README.md and summarize the project"

The filesystem MCP server will handle file operations.

Options & Variations

Command Description Example
/mcp list Show all servers /mcp list
/mcp install <id> Install from catalog /mcp install memory
/mcp start <name> Start a server /mcp start filesystem
/mcp stop <name> Stop a server /mcp stop filesystem
/mcp status <name> Check status /mcp status filesystem
/mcp remove <name> Remove a server /mcp remove filesystem

Troubleshooting

Problem Cause Solution
Server 'X' not found Server not installed or typo Use /mcp list to see available servers
Server stuck in starting state Subprocess failed to start Check /mcp status for error details, try /mcp stop then /mcp start
Server shows error state Configuration issue Check environment variables and command-line arguments
Tools not available after start Agent not reloaded The agent should auto-reload; if not, restart Code Puppy
Quarantined status Server failed repeatedly Use /mcp stop then /mcp start to reset
