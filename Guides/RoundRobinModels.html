How to Use Round-Robin Models

What You'll Learn

By the end of this guide, you'll be able to configure round-robin models to distribute requests across multiple models, helping you overcome rate limits and balance load.

Prerequisites

  • Code Puppy installed and running
  • At least two models configured in your models.json or ~/.code_puppy/models.json
  • Familiarity with Model Configuration

Quick Version

Add a round-robin model to your configuration:

{
  "my_round_robin": {
    "type": "round_robin",
    "models": ["model1", "model2", "model3"],
    "rotate_every": 1
  }
}

Then select it with /model and choose my_round_robin.

Detailed Steps

1. Understand Round-Robin Distribution

Round-robin models cycle through multiple candidate models in sequence. This is useful when:

  • Rate limits: Spread requests across multiple API keys to avoid hitting per-model limits
  • Load balancing: Distribute work across different model providers
  • Cost optimization: Mix expensive and cheaper models
  • Reliability: If one model is temporarily unavailable, others can still serve requests

2. Configure Your Round-Robin Model

Add an entry to your models.json (or ~/.code_puppy/models.json):

{
  "cerebras_round_robin": {
    "type": "round_robin",
    "models": ["qwen1", "qwen2", "qwen3"],
    "rotate_every": 5
  }
}

Configuration options:

Option Description Example
type Must be "round_robin" "round_robin"
models List of model names to cycle through ["model1", "model2"]
rotate_every Number of requests per model before rotating (default: 1) 5
TIP

Each model name in the models list must be a valid model already defined in your configuration. The round-robin model will resolve these to actual model instances.

3. Choose Your Rotation Strategy

The rotate_every parameter controls how many consecutive requests go to each model:

rotate_every = 1 (default):

Request 1 → Model A
Request 2 → Model B
Request 3 → Model C
Request 4 → Model A (cycle repeats)

rotate_every = 5:

Requests 1-5  → Model A
Requests 6-10 → Model B
Requests 11-15 → Model C
Requests 16-20 → Model A (cycle repeats)

When to use each:

Scenario Recommended rotate_every
Maximizing distribution 1
Reducing connection overhead 5-10
Session-based workloads 10-20
Testing specific models Large value (e.g., 100)

4. Select the Round-Robin Model

Use the model picker to select your round-robin configuration:

/model

Tab through the list and select your round-robin model (e.g., cerebras_round_robin).

The model name will display as:

  • round_robin:model1,model2,model3 (default rotation)
  • round_robin:model1,model2,model3:rotate_every=5 (custom rotation)

5. Verify It's Working

Watch the model name in your session. As you make requests, you should see the underlying model change according to your rotation settings.

You can also check the session logs to see which specific model handled each request.

Example: Rate Limit Distribution

Scenario: You have three Cerebras API keys, each with a rate limit of 100 requests/hour. You want to maximize throughput.

Configuration:

{
  "cerebras_key1": {
    "type": "cerebras",
    "name": "zai-glm-4.7",
    "custom_endpoint": {
      "url": "https://api.cerebras.ai/v1",
      "api_key": "$CEREBRAS_API_KEY1"
    },
    "context_length": 131072
  },
  "cerebras_key2": {
    "type": "cerebras",
    "name": "zai-glm-4.7",
    "custom_endpoint": {
      "url": "https://api.cerebras.ai/v1",
      "api_key": "$CEREBRAS_API_KEY2"
    },
    "context_length": 131072
  },
  "cerebras_key3": {
    "type": "cerebras",
    "name": "zai-glm-4.7",
    "custom_endpoint": {
      "url": "https://api.cerebras.ai/v1",
      "api_key": "$CEREBRAS_API_KEY3"
    },
    "context_length": 131072
  },
  "cerebras_round_robin": {
    "type": "round_robin",
    "models": ["cerebras_key1", "cerebras_key2", "cerebras_key3"],
    "rotate_every": 1
  }
}

Result: You can now make up to 300 requests/hour by distributing across all three keys.

Options & Variations

Single Model (Testing)

You can create a round-robin with a single model for testing purposes:

{
  "test_rr": {
    "type": "round_robin",
    "models": ["my_model"],
    "rotate_every": 100
  }
}

This always uses the same model but with a different model name prefix.

Mixing Different Model Types

Round-robin works with any model types:

{
  "mixed_round_robin": {
    "type": "round_robin",
    "models": ["gpt-5.2", "claude-4-5-sonnet", "Gemini-3"],
    "rotate_every": 2
  }
}

Nested Round-Robin

You can nest round-robin models (though this is advanced):

{
  "group1": {
    "type": "round_robin",
    "models": ["model_a", "model_b"],
    "rotate_every": 1
  },
  "group2": {
    "type": "round_robin",
    "models": ["model_c", "model_d"],
    "rotate_every": 1
  },
  "super_rr": {
    "type": "round_robin",
    "models": ["group1", "group2"],
    "rotate_every": 1
  }
}

Troubleshooting

Problem Cause Solution
ValueError: Round-robin model requires a 'models' list Missing or invalid models field Ensure models is a non-empty array of valid model names
rotate_every must be at least 1 Invalid rotation value Set rotate_every to 1 or higher
Model not found error One of the listed models doesn't exist Verify all model names in the models list are defined
Not rotating as expected rotate_every is too high Lower the rotate_every value or make more requests
