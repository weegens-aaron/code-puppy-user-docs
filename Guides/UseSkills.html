How to Use Skills

What You'll Learn

By the end of this guide, you'll understand how to activate skills during agent conversations, when to use them, and how they enhance Code Puppy's capabilities.

Prerequisites

  • Code Puppy installed and running
  • At least one skill installed (see How to Install Skills)
  • Familiarity with basic agent interaction

Quick Version

# Start an agent
/agent code-puppy

# Ask for help with something that matches a skill
# The agent will automatically detect and activate the relevant skill

How Skills Work in Practice

Skills are automatically discovered and dynamically activated by agents. You don't need to manually load them—just ask for help with something that matches a skill's description.

The Skill Activation Flow

flowchart LR
    A[You ask for help] --> B[Agent analyzes request]
    B --> C[Agent finds matching skill]
    C --> D[Agent activates skill]
    D --> E[Agent follows skill instructions]
    E --> F[You get specialized help]

Using Skills with Agents

Step 1: Start an Agent

/agent code-puppy

Step 2: Ask for Help

When you ask for help with something that matches a skill, the agent will automatically:

  1. Detect the relevant skill from its knowledge of available skills
  2. Activate the skill by loading its full instructions
  3. Follow the skill's guidance to help you

Example conversation:

You: Help me create a Dockerfile for this Python app

Agent: I'll help you containerize this Python app. Let me activate the docker skill first.

[Agent internally calls activate_skill(skill_name="docker")]

Agent: Now I have the docker skill loaded. Here's what I recommend...

[Agent follows the docker skill's best practices to create the Dockerfile]

Step 3: See the Results

The agent will use the skill's instructions to provide specialized, high-quality guidance.

Skill Discovery and Activation

Automatic Activation

Agents automatically activate skills when:

  • User request matches a skill's description or tags
  • Task domain aligns with a skill's purpose
  • Specialized knowledge is needed

Example triggers:

User Request Skill Activated
"Help me set up Kubernetes" kubernetes
"Review this for security issues" security-audit
"Write pytest tests" python-testing
"Create a docker-compose file" docker

Manual Skill Activation

If you know which skill you want to use, you can ask the agent to activate it explicitly:

You: Please use the docker skill to help me containerize this app

The agent will then call activate_skill(skill_name="docker") to load the skill's full instructions.

Skill Tools Reference

Agents have access to two dedicated tools for working with skills:

list_or_search_skills

Lists all available skills, optionally filtered by search query.

When agents use it:

  • At the start of a task to see what's available
  • When searching for a skill matching keywords

Example:

# List all skills
list_or_search_skills()

# Search for docker-related skills
list_or_search_skills(query="docker")

Returns:

  • skills: List of skill metadata (name, description, path, tags)
  • total_count: Total number of skills found
  • query: The search query (if provided)

activate_skill

Loads and activates a specific skill by name.

When agents use it:

  • When a user's task matches a skill's description
  • To load full instructions for a specific capability

Example:

activate_skill(skill_name="docker")

Returns:

  • skill_name: Name of the activated skill
  • content: Full SKILL.md content (including instructions)
  • resources: List of available resource files
  • error: Error message (if activation failed)

Example Workflows

Example 1: Docker Containerization

Scenario: You need to containerize a Python web application.

You: Help me create a Dockerfile for this Flask app

Agent: I'll help you create a Dockerfile. Let me check what skills are available.

[Agent calls list_or_search_skills(query="docker")]

Agent: I found the 'docker-expert' skill. Let me activate it.

[Agent calls activate_skill(skill_name="docker-expert")]

Agent: Based on the docker skill's best practices, here's what I recommend:

1. Use a multi-stage build to minimize image size
2. Use python:3.11-slim as the base image
3. Install dependencies in a virtual environment
4. Run as non-root user for security

Here's the Dockerfile:

FROM python:3.11-slim AS builder WORKDIR /app COPY requirements.txt . RUN python -m venv /opt/venv && \ /opt/venv/bin/pip install --no-cache-dir -r requirements.txt

FROM python:3.11-slim WORKDIR /app COPY --from=builder /opt/venv /opt/venv ENV PATH="/opt/venv/bin:$PATH" COPY . . USER nobody EXPOSE 8000 CMD ["python", "app.py"]

Example 2: Security Audit

Scenario: You want to review your code for security vulnerabilities.

You: Can you review this code for security issues?

Agent: Let me activate the security-audit skill to help with this.

[Agent calls activate_skill(skill_name="security-audit")]

Agent: Based on the security audit checklist, I'll check for:

1. SQL injection vulnerabilities
2. Cross-site scripting (XSS) risks
3. Insecure dependencies
4. Hardcoded secrets

[Agent analyzes your code and provides specific findings]

Example 3: Python Testing

Scenario: You need to write comprehensive tests for a Python module.

You: Help me write tests for this Python module

Agent: I'll help you write tests. Let me activate the python-testing skill.

[Agent calls activate_skill(skill_name="python-testing")]

Agent: Following the pytest best practices from the skill, I recommend:

1. Use fixtures for setup/teardown
2. Mock external dependencies
3. Achieve >80% code coverage
4. Use parametrize for edge cases

Here's the test file:

import pytest from unittest.mock import Mock, patch from mymodule import process_data

@pytest.fixture def sample_data(): return {"key": "value"}

def test_process_data_success(sample_data): result = process_data(sample_data) assert result["status"] == "success"

@patch('mymodule.external_api') def test_process_data_with_mock(mock_api, sample_data): mock_api.return_value = {"response": "ok"} result = process_data(sample_data) assert result["api_called"] is True

Tips & Best Practices

TIP

Let the Agent Decide: You don't need to manually activate skills. Just describe what you need, and the agent will find the right skill.

TIP

Be Specific: The more specific your request, the better the agent can match it to the right skill.

TIP

Multiple Skills: Agents can activate multiple skills for complex tasks that span multiple domains.

TIP

Check Available Skills: Run /skills list to see what skills are available before starting a task.

TIP

Skill Quality Matters: Skills with detailed instructions and good metadata produce better results.

Troubleshooting

Problem Cause Solution
"No skills found" Skills integration disabled Run /skills enable
Skill not activating Skill name doesn't match Check /skills list for exact skill names
Agent ignores skills No matching skill Install a skill for that domain
Skill activation fails Missing SKILL.md Ensure skill has valid SKILL.md file
Agent doesn't use skill instructions Skill not properly activated Ask agent to explicitly activate the skill
