Configuration Commands Reference

Overview

This reference covers all configuration-related commands in Code Puppy. These commands let you view and modify the application's behavior, manage sessions, configure colors, and more.

Core Configuration Commands

/show

Display the current puppy configuration in a formatted status view.

Usage:

/show

What it shows:

  • puppy_name - Your puppy's name
  • owner_name - Your name
  • current_agent - Currently active agent
  • default_agent - Default agent for new sessions
  • model - Active model
  • YOLO_MODE - Whether YOLO mode is on/off
  • DBOS - Whether DBOS is enabled
  • auto_save_session - Auto-save status
  • protected_tokens - Recent tokens preserved during compaction
  • compaction_threshold - Context usage percentage that triggers compaction
  • compaction_strategy - Either "summarization" or "truncation"
  • resume_message_count - Messages shown on resume
  • reasoning_effort - OpenAI reasoning effort level
  • verbosity - OpenAI verbosity level
  • temperature - Current temperature (per-model or global)
  • cancel_agent_key - Key binding to cancel agent tasks

Example Output:

🐶 Puppy Status

puppy_name:            code_puppy
owner_name:            Developer
current_agent:         Code Puppy
default_agent:         code_puppy
model:                 claude-3-5-sonnet
YOLO_MODE:             off
DBOS:                  enabled
auto_save_session:     enabled
protected_tokens:      1000 recent tokens preserved
compaction_threshold:  80.0% context usage triggers compaction
compaction_strategy:   summarization
reasoning_effort:      medium
verbosity:             medium
temperature:           0.7
cancel_agent_key:      ctrl+c

/set

Set configuration values by key.

Usage:

/set <key> <value>
/set <key>=<value>

Available Keys:

Key Description Values
yolo_mode Enable/disable YOLO mode true / false
enable_dbos Enable DBOS storage true / false
auto_save_session Auto-save chat after responses true / false
protected_tokens Recent tokens to preserve Number (e.g., 1000)
compaction_threshold Context usage % to trigger compaction Percentage (e.g., 0.8)
compaction_strategy Compaction method summarization / truncation
resume_message_count Messages shown on resume Number (e.g., 10)
cancel_agent_key Key to cancel agent tasks ctrl+c / ctrl+k / ctrl+q

Examples:

/set yolo_mode true
/set auto_save_session=false
/set protected_tokens 2000
/set compaction_strategy truncation
NOTE

Some settings (like enable_dbos and cancel_agent_key) require a restart to take effect.

/reasoning

Set OpenAI reasoning effort level for GPT-5 models.

Usage:

/reasoning <level>

Levels:

  • minimal - Least reasoning
  • low - Low reasoning
  • medium - Medium reasoning (default)
  • high - High reasoning
  • xhigh - Extra high reasoning

Example:

/reasoning high

Result:

Reasoning effort set to 'high' and active agent reloaded

/verbosity

Set OpenAI verbosity level for GPT-5 models.

Usage:

/verbosity <level>

Levels:

  • low - More concise responses
  • medium - Balanced (default)
  • high - More verbose responses

Example:

/verbosity low

Result:

Verbosity set to 'low' and active agent reloaded

/pin_model

Pin a specific model to an agent.

Usage:

/pin_model <agent-name> <model-name>

Examples:

/pin_model code_puppy claude-3-5-sonnet
/pin_model planner gpt-4o

What it does:

  • For built-in Python agents: Stores the pin in config
  • For JSON agents: Modifies the JSON file directly
  • If the current agent is pinned, it reloads immediately

Result:

Model 'claude-3-5-sonnet' pinned to agent 'code_puppy'
Active agent reloaded with pinned model 'claude-3-5-sonnet'

/unpin

Unpin a model from an agent (resets to default).

Usage:

/unpin <agent-name>

Example:

/unpin code_puppy

Result:

Model unpinned from agent 'code_puppy' (reset to default)
Active agent reloaded with default model

/diff

Configure diff highlighting colors for additions and deletions.

Usage:

/diff

Opens an interactive TUI picker to select colors.

Recommended Colors:

Additions (green tones):

  • green, bright_green, chartreuse1, green3, sea_green1

Deletions (red/orange tones):

  • orange1, red, bright_red, indian_red, dark_red

All Available Colors:

  • Basic: black, red, green, yellow, blue, magenta, cyan, white
  • Bright: bright_black, bright_red, bright_green, etc.
  • Special: orange1, orange3, deep_sky_blue1, turquoise2, steel_blue1, chartreuse1, gold1, etc.

Hex/RGB Support: You can also use hex colors like #ff0000 or rgb(255,0,0).

/colors

Configure banner colors for tool outputs (THINKING, SHELL COMMAND, etc.).

Usage:

/colors

Opens an interactive TUI picker to customize banner colors.

Session Management Commands

/session

Show or rotate the autosave session ID.

Usage:

/session          # Show current session ID
/session id       # Same as above
/session new      # Create new session and rotate ID

Example:

/session

Result:

Autosave Session: abc123-def456
Files prefix: ~/.code_puppy/autosave/abc123-def456

/compact

Summarize and compact current chat history using the configured compaction strategy.

Usage:

/compact

What it does:

  • Uses either summarization or truncation based on compaction_strategy config
  • Preserves protected tokens (recent messages)
  • Shows token reduction percentage

Example:

/compact

Result:

🤔 Compacting 50 messages using summarization strategy... (~15000 tokens)
✨ Done! History: 50 → 25 messages via summarization
🏦 Tokens: 15,000 → 6,000 (60.0% reduction)

/truncate

Truncate history to N most recent messages.

Usage:

/truncate <N>

Example:

/truncate 10

Result:

Truncated message history from 50 to 10 messages (keeping system message and 9 most recent)
NOTE

Always keeps the system message plus the N-1 most recent messages.

/resume or /autosave_load

Load an autosave session interactively.

Usage:

/resume
/autosave_load

Opens an interactive menu to select and load a previous autosave session.

/dump_context

Save current message history to a file.

Usage:

/dump_context <session-name>

Example:

/dump_context my-project-context

Result:

✅ Context saved: 100 messages (25000 tokens)
📁 Files: ~/.code_puppy/contexts/my-project-context.pkl, ...metadata.json

/load_context

Load message history from a previously saved context file.

Usage:

/load_context <session-name>

Example:

/load_context my-project-context

Result:

✅ Context loaded: 100 messages (25000 tokens)
📁 From: ~/.code_puppy/contexts/my-project-context.pkl

Autosave session rotated to: new-id-123

MCP Server Commands

/mcp

Manage Model Context Protocol (MCP) servers.

Usage:

/mcp                    # Show status dashboard
/mcp <subcommand>       # Run a specific subcommand

Available Subcommands:

Subcommand Description
list List all MCP servers and their status
start <server> Start a specific server
start-all Start all configured servers
stop <server> Stop a specific server
stop-all Stop all running servers
restart <server> Restart a specific server
status <server> Show detailed status for a server
test <server> Test a server connection
edit <server> Edit a server configuration
remove <server> Remove a server configuration
logs <server> Show server logs
search <query> Search the server catalog
install <server> Install a server from catalog
help Show help information

Examples:

/mcp list
/mcp start filesystem
/mcp status filesystem
/mcp search git
/mcp install server-github

Skills Commands

/skills

Manage agent skills (custom tools and capabilities).

Usage:

/skills                    # Show interactive TUI
/skills <subcommand>       # Run a specific subcommand

Available Subcommands:

Subcommand Description
list List all discovered skills
enable <name> Enable a specific skill
disable <name> Disable a specific skill
toggle Toggle skills integration on/off
refresh Refresh the skill cache
help Show help information

Examples:

/skills
/skills list
/skills enable my-custom-skill
/skills disable unused-skill

Scheduler Commands

/scheduler

Manage scheduled tasks and the scheduler daemon.

Usage:

/scheduler              # Show status
/scheduler <subcommand> # Run a specific subcommand

Available Subcommands:

Subcommand Description
start Start the scheduler daemon
stop Stop the scheduler daemon
status Show daemon and task status
list List all scheduled tasks
run <id> Run a specific task immediately

Examples:

/scheduler start
/scheduler status
/scheduler list
/scheduler run task-123
