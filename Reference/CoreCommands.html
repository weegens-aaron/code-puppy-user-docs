Core Commands Reference

Overview

This reference covers all built-in slash commands available in Code Puppy. These commands are always available and don't require any plugins or additional setup.

Help & Information

/help, /h

Show the complete list of available commands.

Usage:

/help
/h

Output:

Built-in Commands
/help         Show this help message
/agent        Switch to a different agent
...

Custom Commands
/skills       Manage agent skills
/scheduler    Manage scheduled tasks

Shell Pass-through
!<command>    Run a shell command directly (e.g., !git status)
TIP

Custom commands from plugins appear in a separate section.

/tools

Show available tools and capabilities that the agent can use.

Usage:

/tools

Output: Displays a formatted list of tools including:

  • File operations (read, write, edit)
  • Command execution
  • Browser control
  • Agent skills
  • And more

/motd

Show the latest Message of the Day (MOTD) with updates, tips, or announcements.

Usage:

/motd

/tutorial

Run the interactive onboarding tutorial wizard.

Usage:

/tutorial

What happens:

  1. The wizard guides you through setup options
  2. You can choose authentication method (ChatGPT or Claude)
  3. Or skip and configure later

Results:

  • completed - Tutorial finished successfully
  • skipped - You can run /tutorial anytime to restart
  • chatgpt / claude - OAuth flow starts for authentication

Navigation & Filesystem

/cd, /directories

Change the working directory or show current directory structure.

Usage:

/cd              # Show current directory
/cd <path>       # Change to specified directory
/cd ..           # Go up one level
/cd ~/projects   # Use absolute or home-relative paths

Example:

/cd src/code_puppy
> Changed directory to: C:\code_puppy\src\code_puppy\code_puppy
NOTE

When you change directories, the agent automatically reloads to reflect the new project context.

Model & Agent Management

/agent, /a

Switch to a different agent or show available agents.

Usage:

/agent              # Interactive agent picker
/agent <name>       # Switch to specific agent
/a <name>           # Alias for /agent

Example:

/agent code_puppy
> Switched to agent: Code Puppy (General-purpose coding agent)
> Auto-save session rotated to: 20240115-abc123

Available agents include:

  • code_puppy - General-purpose coding agent
  • planning - Task planning and breakdown
  • code_reviewer - Code review and feedback
  • python_programmer - Python-specific development
  • And many language-specific reviewers

/model, /m

Set the active AI model for the current session.

Usage:

/model              # Interactive model picker
/model <name>       # Set specific model
/m <name>           # Alias for /model

Example:

/model claude-opus-4-6
> Active model set and loaded: claude-opus-4-6
TIP

Use /show to see your current model and configuration.

/pin_model

Pin a specific model to an agent (so it always uses that model).

Usage:

/pin_model <agent> <model>

Example:

/pin_model python_programmer claude-opus-4-6
> Model 'claude-opus-4-6' pinned to agent 'python_programmer'

To unpin:

/unpin <agent>

Configuration

/show

Display current Code Puppy configuration in a formatted view.

Usage:

/show

Output includes:

  • puppy_name - Your Code Puppy instance name
  • owner_name - Your configured name
  • current_agent - Currently active agent
  • model - Active AI model
  • YOLO_MODE - Whether dangerous operations are allowed
  • auto_save_session - Auto-save status
  • protected_tokens - Recent tokens preserved during compaction
  • compaction_threshold - Context usage threshold
  • temperature - Model temperature setting
  • And more

/set

Set configuration values dynamically.

Usage:

/set <key> <value>
/set <key>=<value>

Common settings:

Key Values Description
yolo_mode true / false Allow dangerous operations
auto_save_session true / false Enable auto-save
temperature 0.0 - 2.0 Model creativity
cancel_agent_key ctrl+c, ctrl+k, ctrl+q Key to cancel agent
enable_dbos true / false Enable DBOS persistence

Examples:

/set yolo_mode true
/set temperature 0.7
/set cancel_agent_key ctrl+k
WARNING

Some settings (like cancel_agent_key and enable_dbos) require a restart to take effect.

/reasoning

Set OpenAI reasoning effort for GPT-5 models.

Usage:

/reasoning <level>

Levels:

  • minimal - Fastest, least reasoning
  • low - Quick reasoning
  • medium - Balanced (default)
  • high - More thorough reasoning
  • xhigh - Maximum reasoning

Example:

/reasoning high
> Reasoning effort set to 'high' and active agent reloaded

/verbosity

Set OpenAI verbosity for GPT-5 models.

Usage:

/verbosity <level>

Levels:

  • low - More concise responses
  • medium - Balanced (default)
  • high - More detailed responses

Example:

/verbosity low
> Verbosity set to 'low' and active agent reloaded

Session Management

/session, /s

Show or rotate the autosave session ID.

Usage:

/session          # Show current session ID
/session id       # Same as above
/session new      # Create new session and rotate ID
/s                # Alias for /session

Example:

/session
> Autosave Session: 20240115-abc123
> Files prefix: C:\code_puppy\.puppy\autosave\20240115-abc123

/session new
> New autosave session id: 20240115-def456

/compact

Summarize and compact the current chat history to save tokens.

Usage:

/compact

What happens:

  1. Analyzes your message history
  2. Applies configured compaction strategy (summarization or truncation)
  3. Preserves protected recent tokens
  4. Shows token reduction statistics

Example output:

✨ Done! History: 50 → 25 messages via summarization
🏦 Tokens: 45,000 → 18,500 (58.9% reduction)
TIP

Use /compact when you're running low on context window space.

/truncate

Truncate history to keep only the N most recent messages.

Usage:

/truncate <N>

Example:

/truncate 10
> Truncated message history from 50 to 10 messages (keeping system message and 9 most recent)
CAUTION

This permanently removes older messages. Use /compact for safer space-saving.

/resume, /autosave_load

Load a previously autosaved session interactively.

Usage:

/resume
/autosave_load

What happens:

  1. Shows a list of available autosave sessions
  2. You select one to load
  3. Conversation history is restored

/dump_context

Save the current message history to a file for later use.

Usage:

/dump_context <name>

Example:

/dump_context my-project-session
> ✅ Context saved: 45 messages (32,000 tokens)
> 📁 Files: C:\code_puppy\.puppy\contexts\my-project-session.pkl

/load_context

Load a previously saved message history from a file.

Usage:

/load_context <name>

Example:

/load_context my-project-session
> ✅ Context loaded: 45 messages (32,000 tokens)
> 📁 From: C:\code_puppy\.puppy\contexts\my-project-session.pkl

List available contexts:

/load_context
> Available contexts: my-project-session, another-session, ...

Clipboard & Attachments

/paste, /clipboard, /cb

Paste an image from the clipboard into the current conversation.

Usage:

/paste
/clipboard
/cb

What happens:

  1. Checks if there's an image in your clipboard
  2. Attaches it to your next message
  3. Shows confirmation with pending image count

Example:

/paste
> 📋 [image placeholder]
> Total pending clipboard images: 1
> Type your prompt and press Enter to send with the image(s)
TIP

You can paste multiple images before sending. They'll all be included in your next message.

Exit

/exit, /quit

Exit the interactive Code Puppy session.

Usage:

/exit
/quit
