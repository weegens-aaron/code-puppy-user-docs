Environment Variables Reference

Overview

This reference covers all environment variables used by Code Puppy. Environment variables control behavior, authentication, and system configuration.

Directory Configuration

XDG_CONFIG_HOME

Description: Base directory for configuration files.

Default: ~/.code_puppy (if not set)

Example:

export XDG_CONFIG_HOME=/etc/xdg

Effect:

  • Config file: $XDG_CONFIG_HOME/code_puppy/puppy.cfg
  • MCP servers: $XDG_CONFIG_HOME/code_puppy/mcp_servers.json

XDG_DATA_HOME

Description: Base directory for data files.

Default: ~/.code_puppy (if not set)

Example:

export XDG_DATA_HOME=/usr/local/share

Effect:

  • Models config: $XDG_DATA_HOME/code_puppy/models.json
  • Agents: $XDG_DATA_HOME/code_puppy/agents/
  • Skills: $XDG_DATA_HOME/code_puppy/skills/
  • Contexts: $XDG_DATA_HOME/code_puppy/contexts/

XDG_CACHE_HOME

Description: Base directory for cache files.

Default: ~/.code_puppy (if not set)

Example:

export XDG_CACHE_HOME=/tmp/cache

Effect:

  • Autosaves: $XDG_CACHE_HOME/code_puppy/autosaves/

XDG_STATE_HOME

Description: Base directory for state files.

Default: ~/.code_puppy (if not set)

Example:

export XDG_STATE_HOME=/var/lib/code_puppy

Effect:

  • Command history: $XDG_STATE_HOME/code_puppy/command_history.txt

Database Configuration

DBOS_SYSTEM_DATABASE_URL

Description: Database connection URL for DBOS state storage.

Default: sqlite:///~/.code_puppy/dbos_store.sqlite

Example:

export DBOS_SYSTEM_DATABASE_URL=postgresql://user:pass@localhost/code_puppy

HTTP & Proxy Configuration

HTTP_PROXY / http_proxy

Description: HTTP proxy URL for outgoing requests.

Default: Not set

Example:

export HTTP_PROXY=http://proxy.example.com:8080

HTTPS_PROXY / https_proxy

Description: HTTPS proxy URL for outgoing requests.

Default: Not set

Example:

export HTTPS_PROXY=http://proxy.example.com:8080

NO_PROXY

Description: Comma-separated list of hosts to bypass proxy.

Default: Not set

Example:

export NO_PROXY=localhost,127.0.0.1,.example.com

SSL_CERT_FILE

Description: Path to custom SSL certificate bundle.

Default: System default

Example:

export SSL_CERT_FILE=/etc/ssl/certs/custom-ca.pem

CODE_PUPPY_NO_RETRY

Description: Disable automatic retry for failed HTTP requests.

Values: true, false, 1, 0

Default: false (retry enabled)

Example:

export CODE_PUPPY_NO_RETRY=true

UI & Display Configuration

CODE_PUPPY_NO_TUI

Description: Disable text user interface (TUI).

Values: 1, 0

Default: Not set (TUI enabled)

Example:

export CODE_PUPPY_NO_TUI=1

Effect:

  • When set to 1: Runs in non-interactive mode

CODE_PUPPY_NO_COLOR

Description: Disable colored output.

Values: 0, 1

Default: 0 (colors enabled)

Example:

export CODE_PUPPY_NO_COLOR=1

NO_COLOR

Description: Standard no-color flag (also respected).

Values: Any value disables colors

Default: Not set

Example:

export NO_COLOR=1

COLORTERM

Description: Terminal color support level.

Default: System default

Example:

export COLORTERM=truecolor

TERM

Description: Terminal type.

Default: System default

Example:

export TERM=xterm-256color

Browser & Headless Configuration

HEADLESS

Description: Run browser in headless mode.

Values: true, false

Default: false

Example:

export HEADLESS=true

BROWSER_HEADLESS

Description: Browser headless mode setting.

Values: true, false

Default: true

Example:

export BROWSER_HEADLESS=false

CHROMIUM_HEADLESS

Description: Chromium-specific headless mode.

Values: true, false

Default: false

Example:

export CHROMIUM_HEADLESS=true

CI & Automation Configuration

CI

Description: Running in CI environment.

Values: true, false

Default: Not set

Example:

export CI=true

Effect:

  • Disables interactive prompts
  • Adjusts output formatting

CODE_PUPPY_SKIP_TUTORIAL

Description: Skip onboarding tutorial.

Values: Any value skips tutorial

Default: Not set

Example:

export CODE_PUPPY_SKIP_TUTORIAL=1

PYTEST_CURRENT_TEST

Description: Set by pytest during test execution.

Default: Not set (only during tests)

Effect:

  • Detected during test runs
  • Adjusts behavior for testing

Version & Update Configuration

NO_VERSION_UPDATE

Description: Disable automatic version update checks.

Values: true, false, 1, 0

Default: Not set (updates enabled)

Example:

export NO_VERSION_UPDATE=1

DBOS Configuration

DBOS_APP_VERSION

Description: DBOS application version.

Default: Not set

Example:

export DBOS_APP_VERSION=v1.0.0

DBOS_CONDUCTOR_KEY

Description: DBOS conductor key for distributed execution.

Default: Not set

Example:

export DBOS_CONDUCTOR_KEY=your-conductor-key

DBOS_LOG_LEVEL

Description: DBOS logging level.

Values: debug, info, warn, error

Default: Not set

Example:

export DBOS_LOG_LEVEL=debug

Shell Configuration

SHELL

Description: Default shell for command execution.

Default: System default (e.g., /bin/bash)

Example:

export SHELL=/bin/zsh

COMSPEC

Description: Windows command interpreter.

Default: cmd.exe

Example:

export COMSPEC=C:\Windows\System32\powershell.exe

.env File Support

Code Puppy automatically loads .env files from the current working directory.

Priority Order:

  1. Environment variables (highest)
  2. .env file
  3. puppy.cfg (lowest)

**Example .env file:**

# API Keys
OPENAI_API_KEY=sk-your-key
ANTHROPIC_API_KEY=sk-ant-key

# Configuration
XDG_CONFIG_HOME=/etc/xdg

Setting Environment Variables

Temporary (Current Session)

export VARIABLE_NAME=value

Permanent (Shell Profile)

Add to ~/.bashrc, ~/.zshrc, or ~/.profile:

export VARIABLE_NAME=value

Permanent (Windows)

[System.Environment]::SetEnvironmentVariable("VARIABLE_NAME", "value", "User")

Verifying Environment Variables

# Check if set
echo $VARIABLE_NAME

# Check all Code Puppy variables
env | grep CODE_PUPPY
env | grep XDG
env | grep DBOS
