MCP Commands Reference

Overview

This reference covers all /mcp commands for managing Model Context Protocol (MCP) servers in Code Puppy. MCP servers provide additional tools and capabilities to AI agents.

/mcp (no arguments)

Description: Show the MCP server status dashboard.

Usage:

/mcp

/mcp start

Description: Start a specific MCP server.

Usage:

/mcp start <server_name>

Parameters:

Parameter Description
<server_name> Name of the server to start

Examples:

# Start the filesystem server
/mcp start filesystem

Output:

🚀 Starting server: filesystem (subprocess starting in background)
Tip: Use /mcp status to check if the server is fully initialized
Agent reloaded with updated servers

Behavior:

  • stdio servers: Start subprocess asynchronously in the background
  • SSE/HTTP servers: Enable the server (connects on first use)
  • Automatically reloads the agent to pick up new server tools

/mcp edit

Description: Edit an existing MCP server configuration.

Usage:

/mcp edit <server_name>

Parameters:

Parameter Description
<server_name> Name of the server to edit

Examples:

# Edit the filesystem server
/mcp edit filesystem

Behavior:

  • Opens an interactive editor to modify server configuration
  • Allows changing command, arguments, environment variables
  • Saves changes and restarts the server if running

/mcp install

Description: Install an MCP server from the registry.

Usage:

/mcp install <server_id>

Parameters:

Parameter Description
<server_id> Server ID from the registry

Examples:

# Install PostgreSQL server
/mcp install postgres

# Install filesystem server
/mcp install filesystem

Behavior:

  • If no arguments: Opens interactive menu to browse available servers
  • With server_id: Installs the specified server from the registry
  • Prompts for required environment variables and arguments
  • Automatically starts the server after installation

Output:

📦 Installing: PostgreSQL MCP Server
Description: PostgreSQL database MCP server

Required Environment Variables:
  POSTGRES_HOST: localhost
  POSTGRES_PORT: 5432
  POSTGRES_USER: [prompted]
  POSTGRES_PASSWORD: [prompted]

✅ Server installed: postgres
🚀 Starting server: postgres

Server Types

Type Description
STDIO Server runs as a subprocess, communicates via stdin/stdout
SSE Server runs as an HTTP service, communicates via Server-Sent Events
