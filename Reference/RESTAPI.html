REST API Reference

Overview

Code Puppy provides a FastAPI-based REST API for programmatic access to configuration, sessions, commands, and agents. The API also supports WebSocket for real-time communication.

Server Setup

Starting the API Server

# Start the API server
code_puppy api-server

# Or run directly
python -m code_puppy.api.main

Default Address: http://localhost:8765

WebSocket Address: ws://localhost:8765/ws/terminal

Authentication

The API currently runs locally without authentication. For production deployments, consider:

  1. Using a reverse proxy with authentication
  2. Implementing API key authentication
  3. Restricting access to localhost only

Endpoints

Configuration API

GET /api/config/

Description: List all configuration keys and their values.

Response:

{
  "config": {
    "puppy_name": "Code Puppy",
    "owner_name": "John Doe",
    "model": "gpt-5",
    "temperature": "0.7",
    ...
  }
}

GET /api/config/keys

Description: Get list of all valid configuration keys.

Response:

[
  "puppy_name",
  "owner_name",
  "model",
  "temperature",
  ...
]

GET /api/config/{key}

Description: Get a specific configuration value.

Path Parameters:

Name Type Description
key string Configuration key name

Response:

{
  "key": "model",
  "value": "gpt-5"
}

Status Codes:

Code Meaning
200 Success
404 Key not found

PUT /api/config/{key}

Description: Set a configuration value.

Path Parameters:

Name Type Description
key string Configuration key name

Request Body:

{
  "value": "new-value"
}

Response:

{
  "key": "model",
  "value": "gpt-5"
}

Status Codes:

Code Meaning
200 Success
404 Key not found

DELETE /api/config/{key}

Description: Reset a configuration value to default.

Path Parameters:

Name Type Description
key string Configuration key name

Response:

{
  "message": "Config key 'model' reset to default"
}

Sessions API

GET /api/sessions/

Description: List all available sessions.

Response:

[
  {
    "session_id": "abc123",
    "agent_name": "code-puppy",
    "initial_prompt": "Review this code...",
    "created_at": "2024-01-15T10:00:00",
    "last_updated": "2024-01-15T10:30:00",
    "message_count": 15
  },
  ...
]

GET /api/sessions/{session_id}

Description: Get session metadata.

Path Parameters:

Name Type Description
session_id string Session identifier

Response:

{
  "session_id": "abc123",
  "agent_name": "code-puppy",
  "initial_prompt": "Review this code...",
  "created_at": "2024-01-15T10:00:00",
  "last_updated": "2024-01-15T10:30:00",
  "message_count": 15
}

Status Codes:

Code Meaning
200 Success
404 Session not found
504 Timeout reading session

GET /api/sessions/{session_id}/messages

Description: Get the full message history for a session.

Path Parameters:

Name Type Description
session_id string Session identifier

Response:

[
  {
    "role": "user",
    "content": "Review this code...",
    "timestamp": "2024-01-15T10:00:00"
  },
  {
    "role": "assistant",
    "content": "Here's my review...",
    "timestamp": "2024-01-15T10:00:05"
  },
  ...
]

Status Codes:

Code Meaning
200 Success
404 Session messages not found
500 Error loading messages
504 Timeout loading messages

DELETE /api/sessions/{session_id}

Description: Delete a session and its data.

Path Parameters:

Name Type Description
session_id string Session identifier

Response:

{
  "message": "Session 'abc123' deleted"
}

Status Codes:

Code Meaning
200 Success
404 Session not found

Commands API

GET /api/commands/

Description: List available commands.

Response:

{
  "commands": [
    {
      "name": "/think",
      "description": "Enter thinking mode",
      "aliases": []
    },
    {
      "name": "/save",
      "description": "Save current session",
      "aliases": []
    },
    ...
  ]
}

POST /api/commands/execute

Description: Execute a slash command.

Request Body:

{
  "command": "/save my-session"
}

Response:

{
  "success": true,
  "result": "Session saved as 'my-session'"
}

Agents API

GET /api/agents/

Description: List available agents.

Response:

{
  "agents": [
    {
      "name": "code-puppy",
      "description": "Default Code Puppy agent",
      "capabilities": ["code-review", "refactoring", "debugging"]
    },
    ...
  ]
}

POST /api/agents/invoke

Description: Invoke an agent with a prompt.

Request Body:

{
  "agent": "code-puppy",
  "model": "gpt-5",
  "prompt": "Review this code for security issues",
  "working_directory": "/path/to/project"
}

Response:

{
  "session_id": "abc123",
  "status": "started",
  "message": "Agent invocation started"
}

WebSocket API

ws://localhost:8765/ws/terminal

Description: Interactive terminal WebSocket connection.

Connection:

const ws = new WebSocket('ws://localhost:8765/ws/terminal');

ws.onopen = () => {
  console.log('Connected to Code Puppy terminal');
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log('Received:', data);
};

// Send commands
ws.send(JSON.stringify({
  type: 'command',
  command: '/think'
}));

Message Types:

Type Direction Description
command Client → Server Execute a slash command
input Client → Server Send text input
output Server → Client Terminal output
status Server → Client Connection status

Health Check

GET /health

Description: Check API server health.

Response:

{
  "status": "healthy"
}

Documentation

GET /docs

Description: OpenAPI documentation (Swagger UI).

Access: http://localhost:8765/docs

GET /redoc

Description: ReDoc documentation.

Access: http://localhost:8765/redoc

Error Handling

All errors return a consistent format:

{
  "detail": "Error description",
  "error": "error_code"
}

Common Error Codes:

Code HTTP Status Description
not_found 404 Resource not found
timeout 504 Request timed out
internal_error 500 Internal server error

Request Timeout

Default: 30 seconds

Override: Set REQUEST_TIMEOUT in the API configuration.

Exceptions: WebSocket upgrades and streaming endpoints bypass the timeout.

CORS Configuration

The API includes CORS middleware for frontend access:

allow_origins=["*"]  # Local/trusted
allow_credentials=True
allow_methods=["*"]
allow_headers=["*"]

For Production: Restrict allow_origins to specific domains.

PID File

Location: ~/.code_puppy/api_server.pid

Purpose: Tracks running API server process.

Management:

  • Created when server starts
  • Removed when server stops
  • Can be used to check if server is running

Example: Using the API

Python Example

import requests

# Get configuration
response = requests.get('http://localhost:8765/api/config/')
config = response.json()['config']
print(f"Current model: {config.get('model')}")

# Set a configuration value
response = requests.put(
    'http://localhost:8765/api/config/model',
    json={'value': 'gpt-5'}
)
print(response.json())

# List sessions
response = requests.get('http://localhost:8765/api/sessions/')
sessions = response.json()
print(f"Found {len(sessions)} sessions")

# Get session messages
response = requests.get('http://localhost:8765/api/sessions/abc123/messages')
messages = response.json()
for msg in messages:
    print(f"{msg['role']}: {msg['content'][:50]}...")

cURL Example

# Get configuration
curl http://localhost:8765/api/config/

# Set model
curl -X PUT http://localhost:8765/api/config/model \
  -H "Content-Type: application/json" \
  -d '{"value": "gpt-5"}'

# List sessions
curl http://localhost:8765/api/sessions/

# Delete a session
curl -X DELETE http://localhost:8765/api/sessions/abc123
