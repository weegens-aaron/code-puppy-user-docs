Scheduler Commands Reference

Overview

This reference covers all /scheduler commands for managing scheduled tasks in Code Puppy. The scheduler allows you to run automated tasks at regular intervals.

/scheduler (no arguments)

Description: Launch the interactive scheduler wizard to create a new scheduled task.

Usage:

/scheduler

Behavior:

  • Opens an interactive wizard to create a new scheduled task
  • Guides you through: 1. Task name 2. Schedule type (interval, hourly, daily, custom) 3. Agent selection 4. Model selection (optional) 5. Task prompt 6. Working directory

Wizard Steps:

Step Description
1 Enter task name
2 Select schedule type
3 Choose agent
4 Choose model (optional)
5 Enter task prompt
6 Set working directory
7 Review and confirm

Schedule Types:

Option Description
Every 15 minutes Run 4 times per hour
Every 30 minutes Run twice per hour
Every hour Run once per hour
Every 2 hours Run 12 times per day
Every 6 hours Run 4 times per day
Daily Run once per day
Custom interval Specify like 45m, 3h, 2d

Example Output:

============================================================
📅 CREATE NEW SCHEDULED TASK
============================================================

  Task name: Daily Code Review

  Select schedule type:

  ❯ Every hour
      Run once per hour
    Every 2 hours
    Every 6 hours
    Daily
    Custom interval...

  ↑/↓ Navigate  Enter Select  Ctrl+C Cancel

/scheduler start

Description: Start the scheduler daemon in the background.

Usage:

/scheduler start

Output (success):

Starting scheduler daemon...
Scheduler daemon started (PID 12345)

Output (already running):

Scheduler daemon already running (PID 12345)

Output (failure):

Failed to start scheduler daemon

/scheduler stop

Description: Stop the scheduler daemon.

Usage:

/scheduler stop

Description: List all scheduled tasks with details.

Usage:

/scheduler list

      Status: 🔴 disabled

Task Fields:

Field Description
[id] Unique task identifier
Status Enabled/disabled state
Schedule Schedule type and value
Agent Agent that runs the task
Model Model used (or default)
Last run Timestamp and status of last execution

/scheduler run

Description: Run a specific task immediately (outside its schedule).

Usage:

/scheduler run <task-id>

Parameters:

Parameter Description
<task-id> ID of the task to run (from /scheduler list)

Examples:

# Run the daily-code-review task immediately
/scheduler run 1

Output (success):

Running task 1...
Task 'daily-code-review' started successfully

Output (failure):

Running task 999...
Error: Task with ID '999' not found

Related:

  • Guide: How to use round-robin models

Task Configuration

Scheduled tasks are stored in ~/.code_puppy/scheduler/tasks.json.

Task Structure:

{
  "id": "1",
  "name": "Daily Code Review",
  "prompt": "Review the code in the current directory...",
  "agent": "code-puppy",
  "model": "claude-3-5-sonnet",
  "schedule_type": "interval",
  "schedule_value": "24h",
  "working_directory": ".",
  "enabled": true,
  "last_run": "2024-01-15T09:00:00",
  "last_status": "success"
}

Schedule Types

Type Value Format Description
interval 15m, 1h, 2d Run at regular intervals
hourly 1h Run once per hour
daily 24h Run once per day

Interval Examples:

Value Meaning
15m Every 15 minutes
1h Every hour
2h Every 2 hours
6h Every 6 hours
24h Every day
168h Every week

Daemon Management

The scheduler runs as a background daemon process.

Daemon Commands:

Command Description
/scheduler start Start the daemon
/scheduler stop Stop the daemon
/scheduler status Check daemon status

Daemon Location:

  • PID file: ~/.code_puppy/scheduler/daemon.pid
  • Logs: ~/.code_puppy/scheduler/daemon.log
