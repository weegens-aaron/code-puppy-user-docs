Session Commands Reference

Overview

This reference covers all /session and related commands for managing conversation history, autosave sessions, and context files in Code Puppy.

/session

Description: Show or rotate the autosave session ID.

Usage:

/session [id|new]

Subcommands:

Subcommand Description
/session or /session id Display current autosave session ID
/session new Create a new session and rotate the ID

Examples:

# Show current session
/session

Output:

Autosave Session: abc123-def456
Files prefix: code_puppy_autosave_abc123-def456
# Create new session
/session new

Output:

New autosave session id: xyz789-ghi012

/compact

Description: Summarize and compact current chat history using the configured compaction_strategy.

Usage:

/compact

How It Works:

Code Puppy supports two compaction strategies (configured in config.py):

  1. Truncation - Removes older messages while protecting a minimum token count
  2. Summarization - Uses the LLM to summarize older messages into a condensed form

Examples:

# Compact history
/compact

Output:

✨ Done! History: 50 → 25 messages using summarization
🏦 Tokens: 125,000 → 62,500 (50.0% reduction)

Related Configuration:

  • compaction_strategy - Either "truncation" or "summarization"
  • protected_token_count - Minimum tokens to preserve from truncation

Related:

  • Concept: Understanding Agents
  • Reference: Configuration Options

/truncate

Description: Truncate history to N most recent messages (keeps system message + N-1 recent messages).

Usage:

/truncate <N>

Parameters:

Parameter Description
<N> Number of messages to keep (must be positive integer)

Examples:

# Keep system message + 10 most recent messages
/truncate 11

Output:

Truncated message history from 50 to 11 messages (keeping system message and 10 most recent)

Behavior:

  • Always keeps the first message (system prompt)
  • Keeps the N-1 most recent user/assistant messages
  • Discards all messages in between

Related:

  • Guide: How to use slash commands

/autosave_load (alias: /resume)

Description: Load an autosave session interactively.

Usage:

/autosave_load

Aliases:

  • /resume

How It Works:

This command opens an interactive menu showing all available autosave sessions. Select one to resume that conversation.

Examples:

# Open autosave session picker
/autosave_load

Output:

Select an autosave session to resume:
  1. code_puppy_autosave_abc123-def456 (2024-01-15 14:30)
  2. code_puppy_autosave_xyz789-ghi012 (2024-01-15 10:15)
  3. code_puppy_autosave_mno345-pqr678 (2024-01-14 18:45)

Related:

  • Tutorial: Your First Code Task

/dump_context

Description: Save current message history to a context file.

Usage:

/dump_context <name>

Parameters:

Parameter Description
<name> Name for the context file (stored in contexts/ directory)

Examples:

# Save current conversation as "project_review"
/dump_context project_review

Output:

✅ Context saved: 45 messages (98,500 tokens)
📁 Files: contexts/project_review.pkl, contexts/project_review.json

File Format:

  • .pkl - Pickle file with full message history
  • .json - Metadata including message count, token count, timestamp

Related:

  • Guide: How to use slash commands

/load_context

Description: Load message history from a context file.

Usage:

/load_context <name>

Parameters:

Parameter Description
<name> Name of the context file (without extension)

Examples:

# Load previously saved context
/load_context project_review

Output:

✅ Context loaded: 45 messages (98,500 tokens)
📁 From: contexts/project_review.pkl

Autosave session rotated to: new-session-id

Behavior:

  • Loads the message history into the current agent
  • Automatically rotates the autosave session ID to prevent overwriting
  • Displays recent message history for context

Available Contexts: If the specified context doesn't exist, Code Puppy will show available contexts:

Context file not found: contexts/project_review.pkl
Available contexts: project_review, code_analysis, bug_fix_session
