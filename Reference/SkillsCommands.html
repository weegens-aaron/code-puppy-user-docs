Skills Commands Reference

Overview

This reference covers all /skills commands for managing agent skills in Code Puppy. Skills are reusable capabilities that extend what agents can do.

/skills (no arguments)

Description: Launch the interactive skills management TUI.

Usage:

/skills

Behavior:

  • Opens a split-panel interface showing all discovered skills
  • Left panel: Skill list with navigation
  • Right panel: Details for selected skill

Navigation:

Key Action
↑/↓ or j/k Navigate skill list
←/→ Previous/Next page
Enter Toggle skill enabled/disabled
t Toggle skills system on/off
r Refresh skill cache
Ctrl+A Add a skill directory
Ctrl+D Show/manage directories
i Install skills from catalog
q or Escape Exit

Output (example):

 Skills: ENABLED

 > ✓ git_operations
   ✓ file_operations
   ✓ web_scraper
   ✗ database_tools

 Page 1/2

 ↑/↓ or j/k Navigate  ←/→ Page
 Enter Toggle  t Toggle System
 Ctrl+A Add Dir  Ctrl+D Show Dirs
 i Install from catalog  r Refresh  q Exit

/skills list

Description: List all discovered skills in a simple format.

Usage:

/skills list

    Resources: 3

  [enabled] file_operations
    File system operations (read, write, search)
    Resources: 5

  [disabled] database_tools
    Database query and management
    Resources: 2

Related:

  • Guide: How to use skills

/skills enable

Description: Enable a specific skill.

Usage:

/skills enable <skill-name>

Parameters:

Parameter Description
<skill-name> Name of the skill to enable

Examples:

# Enable the git_operations skill
/skills enable git_operations

Output:

Skill 'git_operations' has been enabled.

Related:

  • Guide: How to use skills

/skills disable

Description: Disable a specific skill.

Usage:

/skills disable <skill-name>

Parameters:

Parameter Description
<skill-name> Name of the skill to disable

Examples:

# Disable the database_tools skill
/skills disable database_tools

Output:

Skill 'database_tools' has been disabled.

Related:

  • Guide: How to use skills

/skills toggle

Description: Toggle the entire skills integration system on/off.

Usage:

/skills toggle

Behavior:

  • If skills are enabled: Disables all skills
  • If skills are disabled: Enables all skills

Output (enable):

Skills integration has been enabled.

Output (disable):

Skills integration has been disabled.

Related:

  • Guide: How to use skills

/skills refresh

Description: Refresh the skill cache to discover newly added skills.

Usage:

/skills refresh

Output:

Refreshing skill cache...
Skill cache refreshed successfully.

When to Use:

  • After manually adding new skill files to a skill directory
  • After installing skills from the catalog
  • After modifying skill metadata

Related:

  • Guide: How to install skills

/skills help

Description: Show help information for all skills commands.

Usage:

/skills help

Output:

Available commands:
  /skills                    - Show interactive TUI
  /skills list              - List all skills
  /skills enable <name>     - Enable a skill
  /skills disable <name>    - Disable a skill
  /skills toggle            - Toggle skills integration
  /skills refresh           - Refresh skill cache
  /skills help              - Show this help

Skill Directories

Skills are discovered from the following directories:

Location Description
~/.code_puppy/skills/ Default user skills directory
./skills/ Project-local skills directory
Custom directories Added via /skills TUI (Ctrl+A)

Adding a Custom Directory:

  1. Run /skills to open the TUI
  2. Press Ctrl+A
  3. Enter the directory path
  4. Skills in that directory will be discovered

Managing Directories:

  1. Run /skills to open the TUI
  2. Press Ctrl+D
  3. Enter a number to remove that directory

Skill Status Indicators

Indicator Meaning
✓ Skill is enabled
✗ Skill is disabled
[enabled] Skill is active
[disabled] Skill is inactive

Skill Metadata

Skills should include a SKILL.md file with frontmatter:

---
name: git_operations
description: Git repository operations (clone, commit, push)
tags: [git, version-control, repository]
---

Metadata Fields:

Field Description
name Unique skill identifier
description Human-readable description
tags Categorization tags

Skill Resources

Skills can include resources (files, templates, scripts) that support their functionality. Resources are listed in the skill details panel.
