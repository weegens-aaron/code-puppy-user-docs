Available Tools Reference

Overview

This reference covers all tools available to Code Puppy agents. Tools are the building blocks that enable agents to interact with files, run commands, control browsers, and more.

Tool Categories

Category Description
Agent Tools Invoke and manage sub-agents
File Operations Read, list, and search files
File Modifications Create, edit, and delete files
Command Runner Execute shell commands
Browser Control Automate web browsers
Terminal Tools Manage terminal sessions
Skills Tools Activate and use skills
Scheduler Tools Manage scheduled tasks
Universal Constructor Create custom tools at runtime

Agent Tools

list_agents

Description: List all available agents.

Parameters: None

Returns: List of agent names and descriptions

Example:

agents = list_agents()
# Returns: {'code-puppy': 'Default agent', 'pack-leader': 'Coordinator...'}

invoke_agent

Description: Invoke a sub-agent with a specific prompt.

Parameters:

Name Type Required Description
agent_name string Yes Name of the agent to invoke
prompt string Yes Task description
model string No Model to use (defaults to current)

Returns: Agent response text

Example:

result = invoke_agent(
    agent_name="bloodhound",
    prompt="Find all TODO comments in the codebase",
    model="gpt-5"
)

File Operations

list_files

Description: List files and directories with intelligent filtering.

Parameters:

Name Type Required Default Description
directory string No . Directory to list
recursive boolean No true Include subdirectories

Returns: List of files and directories

Example:

files = list_files(directory="src", recursive=True)

read_file

Description: Read file contents with optional line-range selection.

Parameters:

Name Type Required Default Description
file_path string Yes - Path to the file
start_line integer No null Starting line number
num_lines integer No null Number of lines to read

Returns: File contents

Example:

# Read entire file
content = read_file(file_path="README.md")

# Read specific lines
content = read_file(
    file_path="src/main.py",
    start_line=10,
    num_lines=50
)

grep

Description: Recursively search for text patterns across files.

Parameters:

Name Type Required Default Description
search_string string Yes - Pattern to search (supports regex)
directory string No . Directory to search in

Returns: List of matches with file paths and line numbers

Example:

# Simple search
matches = grep(search_string="TODO", directory="src")

# Regex search
matches = grep(search_string="def\\s+\\w+\\(", directory="src")

File Modifications

create_file

Description: Create a new file or overwrite an existing one.

Parameters:

Name Type Required Default Description
file_path string Yes - Path to the file
content string Yes - File content
overwrite boolean No false Allow overwriting existing files

Returns: Success status

Example:

create_file(
    file_path="docs/README.md",
    content="# Documentation\n\n...",
    overwrite=False
)

replace_in_file

Description: Apply targeted text replacements to an existing file.

Parameters:

Name Type Required Description
file_path string Yes Path to the file
replacements array Yes List of {old_str, new_str} objects

Returns: Success status

Example:

replace_in_file(
    file_path="src/config.py",
    replacements=[
        {"old_str": "DEBUG = False", "new_str": "DEBUG = True"},
        {"old_str": "LOG_LEVEL = 'info'", "new_str": "LOG_LEVEL = 'debug'"}
    ]
)

delete_snippet

Description: Remove the first occurrence of a text snippet from a file.

Parameters:

Name Type Required Description
file_path string Yes Path to the file
snippet string Yes Text to remove

Returns: Success status

Example:

delete_snippet(
    file_path="src/main.py",
    snippet="# Old comment to remove\n"
)

delete_file

Description: Safely delete a file with comprehensive logging.

Parameters:

Name Type Required Description
file_path string Yes Path to the file

Returns: Success status and diff output

Example:

delete_file(file_path="temp/old_file.txt")

Command Runner

agent_run_shell_command

Description: Execute a shell command with monitoring and safety features.

Parameters:

Name Type Required Default Description
command string Yes - Command to execute
cwd string No null Working directory
timeout integer No 60 Timeout in seconds
background boolean No false Run in background

Returns: Command output and status

Example:

# Simple command
result = agent_run_shell_command(command="ls -la")

# With timeout
result = agent_run_shell_command(
    command="npm install",
    timeout=120
)

# Background execution
result = agent_run_shell_command(
    command="npm run dev",
    background=True
)

agent_share_your_reasoning

Description: Request an agent to share its reasoning process.

Parameters:

Name Type Required Description
reasoning string Yes Reasoning to share

Returns: Confirmation

Note: Not available when extended thinking is active for Anthropic models.

Universal Constructor

universal_constructor

Description: Create custom tools at runtime.

Parameters:

Name Type Required Description
tool_name string Yes Name of the new tool

Returns: Tool registration status

Note: Requires enable_universal_constructor to be true in config.

Tool Registration

Tools are registered through the TOOL_REGISTRY dictionary:

TOOL_REGISTRY = {
    "tool_name": register_function,
    ...
}

Plugin Tools: Plugins can register additional tools via the register_tools callback.

Tool Expansions

Some tools expand into multiple tools for backward compatibility:

TOOL_EXPANSIONS = {
    "edit_file": ["create_file", "replace_in_file", "delete_snippet"],
}

When an agent requests edit_file, all expanded tools are registered instead.
