Tutorial: Creating Your First Custom Agent

What You'll Build

By the end of this tutorial, you'll have created a Python Code Reviewer Agent — a specialized assistant that reviews Python code for best practices, bugs, and performance issues. You'll learn the complete workflow from initial concept to a working custom agent.

Time needed: ~15 minutes

You'll need:

  • Code Puppy installed and running
  • A text editor (optional, for editing JSON)
  • Basic familiarity with the /agents command

You should already know:

  • How to switch agents using /agents
  • How to send messages to agents
  • Basic Python programming concepts

The Scenario

You're working on a Python project and want an agent that:

  1. Specializes in Python code review
  2. Focuses on PEP 8 compliance and Pythonic patterns
  3. Checks for common bugs and security issues
  4. Provides concrete refactoring suggestions

Instead of using the general-purpose agents, you'll create a dedicated Python Reviewer that always gives Python-focused feedback.

Step 1: Choose Your Agent Type

Code Puppy supports two agent types:

Type When to Use
JSON Agent Quick setup, static configuration, no custom logic
Python Agent Complex behavior, dynamic tool selection, custom logic

For this tutorial, we'll create a JSON agent first (simplest), then show how to upgrade to Python if needed.

TIP

Start with JSON. You can always migrate to Python later if you need more power.

Step 2: Create the Agent Directory

Open your terminal and create a directory for your agent:

mkdir -p ~/.code_puppy/agents/python-reviewer

What happened: This creates a folder where Code Puppy will look for your agent's configuration.

Step 3: Create the Agent Configuration

Create a file called agent.json inside that directory:

cat > ~/.code_puppy/agents/python-reviewer/agent.json << 'EOF'
{
  "name": "python-reviewer",
  "display_name": "Python Reviewer 🐍",
  "description": "Specialized Python code reviewer for PEP 8, bugs, and best practices",
  "system_prompt": "You are an expert Python developer specializing in code reviews. Your mission is to help developers write clean, Pythonic, and maintainable code.\n\n## Your Expertise\n\n### Code Quality\n- PEP 8 style guide compliance\n- Pythonic patterns and idioms\n- Type hints and type safety\n- Function and class design\n\n### Common Issues\n- Off-by-one errors\n- Mutable default arguments\n- Exception handling anti-patterns\n- Resource leaks (unclosed files, connections)\n- Import organization\n\n### Best Practices\n- DRY (Don't Repeat Yourself)\n- Single Responsibility Principle\n- Meaningful variable names\n- Appropriate use of comprehensions vs loops\n- Generator usage for memory efficiency\n\n## Review Process\n\n1. **Read** the code carefully\n2. **Identify** issues by category (style, bug, performance, security)\n3. **Prioritize** by severity (Critical/High/Medium/Low/Info)\n4. **Explain** why each issue matters\n5. **Suggest** concrete fixes with code examples\n\n## Output Format\n\nFor each finding:\n\n```\n**[SEVERITY]** Issue Title\n- **Location**: function_name or line number\n- **Problem**: Clear description\n- **Why it matters**: Impact explanation\n- **Fix**: Before/after code example\n```\n\n## Tone\n\n- Be constructive, not critical\n- Explain the \"why\" behind recommendations\n- Acknowledge when code is already good\n- Use examples from the codebase when possible\n\nRemember: Great code reviews make developers better, not just code better.",
  "tools": [
    "list_files",
    "read_file",
    "grep",
    "replace_in_file",
    "create_file",
    "agent_run_shell_command"
  ],
  "user_prompt": "What Python code would you like me to review?"
}
EOF

What happened: You've created a complete agent configuration with:

  • A unique name (python-reviewer)
  • A friendly display name with emoji
  • A detailed system prompt defining expertise and behavior
  • Tools needed for code review tasks
  • A custom user prompt
NOTE

The system prompt uses \n for newlines because JSON doesn't support multi-line strings.

Step 4: Verify the JSON Syntax

Before testing, validate your JSON:

cat ~/.code_puppy/agents/python-reviewer/agent.json | python -m json.tool > /dev/null && echo "✅ Valid JSON" || echo "❌ Invalid JSON"

Expected result:

✅ Valid JSON

If you see ❌ Invalid JSON, check for:

  • Missing commas between fields
  • Unescaped quotes inside strings
  • Trailing commas (not allowed in JSON)

Step 5: Load Your Agent

Restart Code Puppy to load your new agent:

# If already running, exit with Ctrl+C or /quit
# Then start again
code_puppy

What happened: Code Puppy scans ~/.code_puppy/agents/ at startup and registers any valid agent configurations.

Step 6: Switch to Your Agent

Use the /agents command to see your new agent:

/agents

You should see Python Reviewer 🐍 in the list. Select it to switch.

Alternative: Use the direct command:

/switch-agent python-reviewer

What happened: You're now talking to your custom agent instead of the default agent.

Step 7: Test Your Agent

Try a simple code review request:

Here's some Python code. Can you review it?

def process_items(items):
    result = []
    for i in range(len(items)):
        if items[i] > 5:
            result.append(items[i] * 2)
    return result

Expected result: Your agent should respond with feedback like:

**[Medium]** Use enumerate instead of range(len())
- **Location**: process_items function
- **Problem**: Using `range(len(items))` is less Pythonic
- **Why it matters**: `enumerate()` is more readable and efficient
- **Fix**:

for i, item in enumerate(items): if item > 5: result.append(item * 2)


**[Info]** Consider list comprehension
- **Location**: process_items function
- **Problem**: Simple filtering/transformation can use comprehension
- **Fix**:

return [item * 2 for item in items if item > 5]

TIP

If your agent doesn't respond as expected, check that: 1. The system_prompt field is correctly formatted 2. Code Puppy was restarted after creating the agent 3. The agent name matches exactly (python-reviewer)

Step 8: Test Tool Access

Your agent has file tools. Try this:

Can you list the Python files in the current directory and review the first one?

Expected result: Your agent should:

  1. Use the list_files tool to find .py files
  2. Use read_file to read the first file
  3. Provide a code review

If tools don't work, verify the tools array in your agent.json includes the correct tool names.

Step 9: Customize Further (Optional)

Pin a Specific Model

You might want your Python reviewer to always use a specific model. Edit agent.json:

{
  "name": "python-reviewer",
  ...
  "pinned_model": "claude-3-5-sonnet"
}

Then restart Code Puppy.

Add More Specialization

Expand the system_prompt with domain-specific rules:

"system_prompt": "... Your expertise includes:\n\n### Django Best Practices\n- Use `select_related` for foreign keys\n- Avoid N+1 queries\n- Follow Django REST framework patterns\n..."

Advanced: Migrate to a Python Agent

If you need dynamic behavior (e.g., different tools based on context), migrate to Python:

Step A: Create the Python File

cat > code_puppy/agents/agent_python_reviewer.py << 'EOF'
"""Python code reviewer agent with dynamic tool selection."""

from .base_agent import BaseAgent


class PythonReviewerAgent(BaseAgent):
    """Specialized Python code reviewer."""

    @property
    def name(self) -> str:
        return "python-reviewer"

    @property
    def display_name(self) -> str:
        return "Python Reviewer 🐍"

    @property
    def description(self) -> str:
        return "Expert Python code reviewer for PEP 8, bugs, and best practices"

    def get_system_prompt(self) -> str:
        return """
You are an expert Python developer specializing in code reviews.

[Copy the system_prompt from your JSON agent here]
"""

    def get_available_tools(self) -> list[str]:
        return [
            "list_files",
            "read_file",
            "grep",
            "replace_in_file",
            "create_file",
            "agent_run_shell_command",
        ]

    def get_user_prompt(self) -> str:
        return "What Python code would you like me to review?"
EOF

What happened: You've created a Python agent that implements the same behavior as your JSON agent but with full programmatic control.

Step B: Remove the JSON Agent

rm -rf ~/.code_puppy/agents/python-reviewer

Step C: Restart Code Puppy

# Exit and restart
code_puppy

Your Python agent is now loaded and ready to use.

Final Result

You now have a working custom agent that:

  • ✅ Specializes in Python code review
  • ✅ Provides structured, actionable feedback
  • ✅ Has access to file tools for reviewing codebases
  • ✅ Can be extended with more rules or migrated to Python

What You Learned

  • JSON agents are the quickest way to create custom agents
  • System prompts define your agent's expertise and behavior
  • Tools give your agent capabilities to interact with files and systems
  • Python agents provide dynamic control when you need it
  • Agent discovery happens at startup, so restart is required after changes
